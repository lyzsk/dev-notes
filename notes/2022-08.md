编译 compile 和 解释 interpret 的区别：

以及跨平台原理： (跨平台指的是 编译后的文件 是否跨平台)

解释型语言： R, Python, Java

编译型语言： C, C++

Compile: 把整个程序源代码，翻译成另一种代码，然后等待被执行，在执行之前，产物是 [另一份代码]

Interpret: 把程序源代码 一行一行的 读懂后执行，在运行时，产物是 [运行结果]

比如

    .py 是被 python.exe 作为解释器一行一行解释，有 bug 的时候，直到 bug 位置才会报错

    而.java 是被 javac.exe 先编译成.class 文件，然后再被 java.exe 执行/翻译 成 JVM 可执行的字节码，
    JVM 动态的将字节码一行一行的执行成 当前操作系统 可执行的 文件格式，所以说 java 也是一种 解释型语言
    JAVA 的特色就是，javac, java 都在 JDK 里，可以做到一次编译，到处运行 -> 即 跨平台

    .c 的跨平台原理： .c 通过 windows compiler 编译后，再到对应平台上运行,
    .c 想在其他平台运行，需要 .c -> 某平台 compiler -> machine code -> 某平台 processor 最终执行
    所以某种意义上 C 语言不是 跨平台的
    同时，C 语言效率比 JAVA 高， 因为他通过编译器产生的可执行文件可以在该平台上直接运行

# Git

    本地结构：
    工作区 -> git add -> 暂存区 -> git commit -> 本地库

    git status	查看状态，防止忘记是否有文件add了但是没commit

    git log	自动分页的时候， space 下一页， b 上一页， end q 退出
    git log --oneline 单行查看


    git reflog 更高级点，可以看head， 也方便 git reset --hard [commit索引的哈希值] 的操作，复制粘贴少

    git reset --hard [索引]	工作区，暂存区，本地库 同时移动
    git reset --mixed [索引]	工作区 不移动; 暂存区，本地库 同时移动
    git reset --soft [索引]	工作区,暂存区 不移动; 本地库 单独移动
    实际上用的多的就是--hard

    rm 删除工作区内文件后，commit到本地库，实际上就是将历史版本回退到添加文件前一个版本

    通过 git diff [文件名]	可以发现，git 实际上是按行管理的， -原内容 再 +原内容和新内容

    git branch -v 	分支版本查询
    git branch [name]	创建分支
    git checkout [branch name]	切换分支

    分支合并冲突例子：
    	master commit 	内容： “添加内容 by master”
    	branch01 commit 内容： “添加内容 by branch01”
    	如果此时 git checkout master 切换到master分支后， git merge branch01 后 会报错：
    		merge conflict
    解决： 人为modify更改源文件后，用commit解决，最终得到整合的master branch
    注意点：	此时commit的时候 不可以使用 git commit -m "comments" [文件名] 的方法
    		应该使用 git commit -m "comments" 不加文件名的方法

    查看远程库别名
    git remote -v

    起别名
    git remote add [别名] [远程库的HTTPS链接]

    推送
    git push [别名] [分支]
    别名一般取 origin 比较好记，忘了也用git remote -v查看

    add commit到 本地库后，
    git push [别名] [分支] 推送到远程库

    关于 为什么第二次push可以直接push不需要验证，因为是在同一台电脑上
    查看Credential Manager可以查看到github的凭据

    pull 相当于 fetch + merge

    fetch仅仅只是 只读的操作，所以不会对本地仓库有所修改
    git fetch [别名] [分支]
    可以检查远程分支
    git checkout -b [本地分支] [别名]/[远程分支名]

    fetch之后再merge
    merge前要记得切换回 git checkout [本地分支名]
    git merge [别名]/[分支名]


    之前用的都是HTTPS地址
    实际，用SSH更方便
    SSH有免密操作：
    	1. 切换到主目录
    	cd ~

    	2.	生成.ssh	注意C要大写
    	ssh-keygen -t rsa -C [github账号邮箱]

    	3. 通过notepad 打开.ssh 里的id_rsa.pub 复制内容
    	4. 打开github账号，通过 setting - SSH and GPG keys 设置密钥
    	6. 生成SSH密钥以后就可以正常push了
    	例子： 	git remote add origin_ssh [远程仓库ssh地址]

    			然后再次确认别名创建成功
    			git remote -v

    			然后add, commit, 然后 git push origin_ssh [分支名]
    			此过程免密

    	SSH免密的好处：	不用每次进行身份验证
    			坏处：	只能针对一个账号

## overflow 属性

规定当内容溢出元素框时发生的事

`visible:` 默认值, 内容不会被修剪会溢出元素框

`hidden:` 内容会被修剪, 并且修建部分不可见

`scroll:` 内容会被修剪, 但是会提供滚动条

`auto:` 如果内容被修剪, 则会提供滚动条

`inherit:` 继承父元素的 overflow 属性

## sroll-snap-type 优化滚动

scroll-snap-type 定义在滚动容器中的一个临时点(snap point)

```

scroll-snap-type: none | [ x | y | block | inline | both ] [ mandatory | proximity ]?

```

`mandatory` 强制滚动, 滚动结束后, 强制滚动停止在规定的位置 比较常用

`proximity` 临近滚动, 滚动结束后, 滚动停止点可能是滚动停止的地方, 也可能再进行额外移动然后停在指定的位置 没用过

tips : `scroll-snap-type: both mandatory` 可以一次性设置 x y 滚动

`scroll-snap-align: start | center | end;` 可以简单的控制将要聚焦的当前滚动子元素在滚动方向上相对于父容器的对齐方式

没用过 ：

-   `scroll-padding` 是作用于滚动父容器，类似于盒子的 padding

-   `scroll-margin` 是作用于滚动子元素，每个子元素的 scroll-margin 可以设置为不一样的值，类似于盒子的 margin

@see https://m.html.cn/web/css/115569633900.html

### 滚动贴合

就是用`mandatory`沿 y 轴强制滚动

```

    /*
    mandatory沿y轴强制滚动
    需要把滚动条设置到父容器, scroll-snap-type才能生效
    溢出时 y轴保留滚动条, x轴隐藏滚动条
    */
    scroll-snap-type: y mandatory;
    overflow-y: scroll;
    overflow-x: hidden;

```

## position 属性

`position: relative` : 生成相对定位的元素，相对于其正常位置进行定位。
因此，"left:20" 会向元素的 LEFT 位置添加 20 像素。

`position: absolute` : 生成绝对定位的元素，相对于 static 定位以外的第一个父元素进行定位。
元素的位置通过 "left", "top", "right" 以及 "bottom" 属性进行规定。

`position: fixed` : 生成绝对定位的元素，相对于浏览器窗口进行定位。
元素的位置通过 "left", "top", "right" 以及 "bottom" 属性进行规定。

`position: static` : 默认值。没有定位，元素出现在正常的流中（忽略 top, bottom, left, right 或者 z-index 声明）。

## white-space 属性

设置如何处理元素内的空白。

这个属性声明建立布局过程中如何处理元素中的空白符。

`normal` : 默认。空白会被浏览器忽略。

`pre` : 空白会被浏览器保留。其行为方式类似 HTML 中的 \<pre> 标签。

`nowrap` : 文本不会换行，文本会在在同一行上继续，直到遇到 \<br> 标签为止。

`pre-wrap` : 保留空白符序列，但是正常地进行换行。

`pre-line` : 合并空白符序列，但是保留换行符。

`inherit` : 规定应该从父元素继承 white-space 属性的值。

<br>

## text-decoration 属性

规定添加到文本的修饰。

`none` 默认。定义标准的文本。

`underline` 定义文本下的一条线。

`overline` 定义文本上的一条线。

`line-through` 定义穿过文本下的一条线。

`blink` 定义闪烁的文本。

min() 函数让你可以从一个逗号分隔的表达式列表中选择最小的值作为属性的值。

min() 函数方法接受一个或多个用逗号分隔的表达式作为他的参数，数值最小的表达式的值将会作为指定的属性的值。

## float 属性

float 属性定义元素在哪个方向浮动。以往这个属性总应用于图像，使文本围绕在图像周围，不过在 CSS 中，任何元素都可以浮动。浮动元素会生成一个块级框，而不论它本身是何种元素。

如果浮动非替换元素，则要指定一个明确的宽度；否则，它们会尽可能地窄。

可能的值:

`left` 元素向左浮动。

`right` 元素向右浮动。

`none` 默认值。元素不浮动，并会显示在其在文本中出现的位置。

`inherit` 规定应该从父元素继承 float 属性的值。

<br>

## 选择器

~（波浪号）：A ~ B 表示选择 A 标签后的所有 B 标签，但是 A 和 B 标签必须有相同的父元素。

~（加号）加号又被称作兄弟选择器。A+B 表示选择紧邻在 A 后面的 B 元素，且 A 和 B 必须拥有相同的父元素，所选到的仅为一个 B 元素标签。

\> (大于号) 表示某个元素的下一代元素。A>B 指选择 A 元素里面的 B 元素，其中 B 元素是 A 元素的第一代。

## vw, vh 和 % 的区别

-   % 是基于 父元素 的宽度/高度百分比

    vw vh 是基于 视窗 的宽度/高度百分比

-   vh 能直接获取高度

    % 在没有设置 `body` 高度情况下, 无法正确获取可视区域高度

-   `vmin` vw vh 中较小值

-   `vman` vw vh 中较大值

问题 : 不同 font-awesome icon 不同 size 解决办法:

解决 : 在 url 末尾加上 ` fa-fw`

要求 : 4.0 以上版本

@see https://fontawesome.com/docs/web/style/fixed-width

问题 : font-awesome 的 i 标签加了 fa-fw 后, 在 collpase toggle 折叠切换后, 依然没法实现同一个 size

调试 : 给 i 标签加 `border: 1px solid #fff;` 发现依然没办法一个 size

解决 : 在 i 标签外套一层 span 标签

例子 ：

```
<li class="navibar-item">
    <span>
        <i class="fa-solid fa-house fa-fw" id="home"></i>
    </span>
    <span class="navibar-item-text">Home</span>
</li>
```

<hr>

问题:

```
body {
    width: 100%;
    height: 100%;
}
```

导致无法填充满页面

解决 :

```
body {
    width: 100vw;
    height: 100vh;
}
```

或者加个 html

```
html,
body {
    width: 100%;
    height: 100%;
}
```

总结: 尽量用 `vw`, `vh`

<br>

问题 : input 标签搜索框自定义提示

解决 :

```
<input
    type="search"
    class="search-text"
    required
    placeholder="Search"
    oninvalid="this.setCustomValidity('Can\' be empty')"
    oninput="setCustomValidity('')"
/>
```

注意 oninput=""后面是不带`this`的, 为了让在提示后, 成功输入 valid 值后自动隐藏提示框
