# 基础信息 (basic info)

| 字段             | 填写指引 / 示例                                                                                                                                                                                                                                                                                                |
| :--------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 需求名称         | 动词 + 名词。<br> _示例：“封装段 Die Attach 胶水有效期预警”_                                                                                                                                                                                                                                                   |
| 所属环节         | ☐ IC 设计 / ☐ 晶圆制造 / ☐ 封装测试 / ☐ 厂务设施                                                                                                                                                                                                                                                               |
| 提出部门 / 人    | PIE/PE/EE/QA + 联系人                                                                                                                                                                                                                                                                                          |
| 应用场景         | 岗位 + 触发时机 + 关键动作 <br> _示例：“PIE/CP 工程师，在每日晨会分析前日低良批次时，或收到 SPC OOC 告警时；选择产品型号和时间范围，查看特定 Bin（如 Bin9 Open）的空间分布趋势，判断是否为机台 Chamber 相关性问题。”_                                                                                          |
| 想解决什么问题？| 为了[业务目标]，目前[现状 / 痛点]，希望系统能[核心能力]。<br> _示例：“为了快速定位 CP 测试良率下降的根因，目前工程师需手动从 MES 导出 5 个 Lot 的 Bin Map 再用 Excel 画图，耗时 2 小时且无法叠加对比；希望系统能按 Product/Recipe 自动聚合多批次 Wafer Map，一键生成 Defect Trend Chart 并高亮共性缺陷区域。”_ |
| 期望上线时间     | UAT 上线时间 / PROD 上线时间                                                                                                                                                                                                                                                                                   |

# 数据与对象模型 (data & object)

| 字段        | 业务填写内容                                                                        |
| :---------- | :---------------------------------------------------------------------------------- |
| 核心对象    | 涉及哪些对象？( 如：Lot, Wafer, Die, Recipe, Equipment, Mask, GDS, Bin Map)         |
| 输入数据源  | 数据从哪来？（手工录入 / 机台 SECS-GEM/ERP/MES/ 文件）                              |
| 关键属性    | 需要记录该对象的哪些具体参数？(如：Recipe Version, Gas Flow Rate, CPK 值, BOM 版本) |
| 输出 / 展示 | 需要看什么图表？导出什么格式？(如：Trend Chart, Pareto 图, CSV, PDF Report)         |
| 数据保留期  | 数据需要存多久？（合规 / 追溯要求）                                                 |

# 业务规则与逻辑 (business rules)

| 规则类型 | 业务描述示例                                                  | 备注                 |
| :------- | :------------------------------------------------------------ | :------------------- |
| 校验规则 | “当 CPK < 1.33 时，禁止该 Lot 流转到下一工序”                 | Validation Logic     |
| 计算逻辑 | “良率 = (Pass Die 数 / Total Die 数) × 100%，需剔除 Edge Die” | Formula              |
| 权限规则 | “只有 QA 经理以上级别才能 Release 被 Hold 的 Lot”             | RBAC Policy          |
| 告警阈值 | “温度连续 3 点超出 Spec ±2σ 触发 OCAP”                        | Alarm/SPC Rule       |
| 联动动作 | “审批通过后，自动调用 MES 接口下发 Recipe”                    | Workflow Integration |
