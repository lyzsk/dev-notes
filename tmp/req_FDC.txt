# 系统设置
## 登录/登出
- 人员登录验证
## 菜单管理
- 菜单自动隐藏/显示，能够根据用户的喜好，设置菜单的隐藏和展示
## 个人信息
- 用户个人信息展示，展示用户的个人信息
## 安全维护
- 用户密码修改，支持用户修改个人密码信息
## 帮助文档
- 支持用户打开帮助文档
## 多语言
- 多语言，默认支持用户选择语言（中文/英文）
# 用户管理
## 用户信息维护
- 用户信息管理，支持在FAB的部门下输入/导入用户信息
## 用户组权限维护
- 用户组设置，支持在用户部门下添加用户组，以及关联用户和设备，即设定用户组权限
## 用户层级维护
- 用户层级设置，根据车间管理岗位，设置用户的从属关系，以及区域的关系
# 设备管理
## 工厂信息维护
- FAB管理，支持用户输入/导入FAB信息，支持多个FAB同时使用，各不影响
## AREA信息维护
- 区域管理，支持用户根据车间输入/导入区域信息
## 部门信息维护
- 部门管理，支持用户输入/导入部门信息
## 设备类型维护
- 设备类型管理，支持用户在区域下输入/导入设备类型信息，以及设置设备类型的腔体信息
## 设备信息维护
- 设备管理，支持用户在设备类型下输入/导入设备信息，并且启用/暂停设备的腔体信息
## 邮箱组配置
- 设备邮件管理，能够根据部门，建立邮件组，并且关联用户以及设备，在后续推送邮件使用
# 参数采集设置
## 参数管理
- 支持按照设备/腔体的SVID/DVID的导入导出
- 并且支持同类型设备的SVID/DVID的相互同步
## 模板管理
- 根据设备类型建立通用的参数管理模板，并且能够绑定到设备
- 支持多腔体设备共用同一模板，也可以用不同的模板
- 支持添加虚拟参数，并且虚拟参数可以通过图形化的组合，支持常用的算数表达式，支持虚拟参数的模拟
- 支持KEY Parameter的设定
## 模板&腔体绑定
- 支持腔体与模板的绑定关系
- 支持绑定后，腔体的SVID自动同步
- 支持绑定后，虚拟参数的SVID自动与该腔体的SVID绑定
- 对绑定的数据进行校验
- 支持KEY Parameter的验证功能
## 事件管理
- 支持按照设备类型输入\导入机台的所有事件
- 设置设备类型Process Event的类型
- 设定设备事件与Process Event的绑定关系
## 维度信息维护
- 支持用户添加/删除/修改Context信息
- 支持常用的Context信息，如Lot ID ,Carrier ID ,Slot ID ,wafer ID ,Stage ,Step, Recipe ID ,Product ID等信息
- 支持用户导入配方信息，以及配方的步骤信息
## 参数计划配置
- 支持根据Process Type建立采集计划
- 支持收集Facility Data
- 支持与附属设备（external sensor）连接并收集数据
- 支持收集Trace Data, Event Data, Alarm Data
- 不同Sensor或不同Step可以设置不同的采样频率
- 灵活的定义数据收集计划 (data collection plan) ,可根据事件或时间或数据点的条件来设置数据收集起始点和结束点
- None Wafer data, Non Process data收集
## 通信协议
- 支持SECS/GEM协议收集数据
- 支持InterfaceA协议收集数据
- 支持OPC协议收集数据
- 支持从File，DB，FTP, TCP/IP中收集 Extra Sensor 数据
## 设备通信模式
- 支持Polling方式收集数据
- 支持Trace方式收集数据
## 数据上报监控
- 在设定的时间段内，没有采集到设备数据自动报警
## PT 中间件
- 能够提供PT，支持EAP的数据与FDC的数据进行分流
## 系统对接
- 支持与EAP或者MES系统对接，获取设备当前作业Context信息
# 告警规则管理
## 告警规则设置
- 支持用户添加，修改 以及导入告警规则，告警规则的设定符合WECO标准
## 告警定义
- 根据告警规则，设定Warning， Critical，以及Outlier的报警方式
## 数据质量报警管理
- 根据设备类型，设置每一个设备类型的数据的采集的数据质量报警，在没有达到采集的点数时，需要进行报警的推送
# 模型管理
## UVA模型创建
- 根据设备类型所对的设备的Process Type，建立UVA模型，并且能够根据根据Context信息，设置UVA的条件，根据Context条件设置数据过滤条件
- 支持Context Group的概念，即类似的Context可以在一个UVA的模型下进行分组，可以设置Context的优先级
- 提供UVA 主页面，能够展示用户最近打开的UVA模型
## Window 窗口配置
- 可以支持模型按照设备RUN按照工艺步骤，以及其他条件分段建立模型
- 支持按照段设置过滤条件，如Context条件设置，以及参数的范围设置，以及根据Step设置
## 数据过滤
- 支持SV，DV，用户自定义过滤Limit，以及用户自定义条件来进行数据过滤
- 支持按照 时间、Count 进行前/后数据过滤
## 模型配置
- 基于单变量模型的设备和工艺健康状况监控
- 具备同时运行多个基于单变量的监控模式能力
- 历史数据模拟
## UVA 模型管理
- 对模型的控制窗口进行统一管理，可以进行集中修改，并且需要保存修改历史能够对模型进行快速复制
- 能够对模型的修改履历进行查看
- 能够对模型不同版本的对比功能
## MVA 模型管理
- 支持基于多变量模型,监控机台和工艺流程的质量.
- 支持提供 PCA、Q 统计量、Hoteling T2、 Diagonal HT2 等 多种算法
- 支持通过数据进行模拟
# 报表分析
## 批次履历
- 以Lot/Wafer 为单位查看设备加工履历，以及查看加工过程中参数的趋势变化并结合模型数据进行分析
## 设备参数履历
- 以参数的维度，进行Lot之间的参数分析，机台/腔体见的分析
- 不同chamber/tool 可放在同一张FDC chart 查看
## 实时数据
- 在趋势图表中显示跟踪参数的实时值，并提供牛眼图表和计量图，支持查看跟踪参数的实时值是否偏移控制限制或规格限制，或 是否保持在控制限制内
## 参数验证
- 对用户指定的参数履历数据，手动进行汇总计算（Min/Max/Count.. ）
## UVA模型履历
- 根据时间查看FD模型中所有Stats的Chart图形的变化趋势，并且可以确认计算每个Stats的Raw Data.
- 可从汇总数据Drilldown到原始数据
## 告警汇总
- 展示根据设备，配方以及统计值分组的UVA结果，统计不同机台每天报警次数Top N 的模型，可以按照设备类型，设备，批次，参数等各种条件，统计报警的数量，能够通过柱状图进行分析，并且能够展示出报警的详细信息
## 设备状态
- 查看工厂的机台状态，并且能够实时的查看机台的加工信息
## 数据质量
- 反应用户选择某台的一个RUN或者多个RUN的Data Quality是否在用户设定的阈值之内，如果Data Quality异常，则说明采集的RUN的样本点不具备FD的意义,需要跟踪原因，进行分析。
## UVA模型统计
- 为每一个设备或者Model计算 CP, CA, and CAP，并且仅仅反馈某个设备的CP小于阈值
## 报表自定义
- 用户根据自己的喜好，设置报表收藏夹功能，可以设定报表产生的条件，按照自定义的时间点，推送FDC相关报表
# 其他
## 软件升级
- 软件升级和新功能发布不影响系统使用 (不停机)
## 服务监听
- 提供Service监控管理工具
## 运行环境
- 支持运行在Linux平台，如Centos/Redhat
- 支持集群化的部署环境
- 支持MQ集群方式进行消息的转发与各模块的通信
## 图标展示工具
- 浏览数据时可放大和缩小, 并显示数据来源, 当鼠标在图表上可以显示Context的数据
- 提供在数据或图标上标记注释
- 可根据product, process flow，recipe，mask ID等Context信息来筛选数据
- 支持双Y轴显示
- 支持overlay显示Trace Data，如overlay by Lot ID，Wafer ID，Tool Name
- 为不同系列数据选择颜色
- 支持多图同屏显示
- 可以添加显示辅助线，如alarm和event的辅助线
- 调整Y轴的刻度
- 支持修改图表显示设定，例如显示图例，显示单位，设置数据点大小，设置以点或者线的形式显示
- 具备导出数据到Excel的功能
- 可以设置某个run的数据highlight显示
- 按时间顺序显示多个run的数据，支持把X轴中间Idle的时间段去掉
- 支持把csv数据文件导入到FDC图表工具中进行图形化展示和分析
