### RTD

#### 仅上扬独有的功能(未纳入交集)

1. 基础数据管理: 区域管理, 派工时效管理, 派工配置管理, 机台派工状态手动开关.
2. 派工规则配置: 配置派工先后顺序的规则 (赛美特的 Macro/Report 编辑对象为通用开发对象, 未明确对应派工规则本身, 仅在版本与发布层面纳入交集).
3. 进度管理: 派工监控, 插单, 替换, 删单, 派工顺序调整.
4. Full Auto 场景: 基础场景与专项场景 (Sorter, Pilot Run, Super-Hot Lot 等).
5. Multi Lot 约束.
6. 全局派工因素 (Critical Ratio, Move Target, Q-time Urgency, Cycle Time 等).
7. Q-time 管理与报警, Q-time Loop 监控.
8. 设备派工优先调度 (同 PPID 优先, 同组内机台优先).
9. 补充需求: Code 高可用, 高并发处理, 多选/全选设备启停.
10. Litho/WET Scheduling 策略集 (全面状态考量, Loading Balance, Chuck 切换, Reticle 连续性/预搬送, 温控 Setup Cost, Litho Dedication, 小片数避让, Batch Size 检查, WET 瓶颈 Tank).
11. 多系统集成: MCS 搬运任务下发, MES (WIP 清单/机台主数据/批次/物料库存/耗材), EMS (机台主数据/台账).
12. 数据同步支持选定列同步.
13. 在线加列不影响系统运行.
14. 不弱于 Oracle OGG 的实时数据同步.
15. 负载均衡, 缓存优先, 自动编译.
16. 系统级版本控制与回退, 事务及异常处理.

#### 仅赛美特独有的功能(未纳入交集)

1. 角色复制 (快速创建相似角色).
2. 用户密码重置.
3. Report 管理: 分组维护, 可视化编辑器, 保存升版, 导入导出, DV 上线/下线, 权限设置与切换分组, 查询.
4. Macro 管理: 可视化编辑器, 导入导出 (版本升版与 DV 发布部分已纳入交集).
5. 机台映射: 批量导入导出与编辑.
6. 全局变量: 批量导入导出与编辑.
7. 依赖关系查询 (变更影响分析).
8. DV 管理: DV 查询删除, 参数表查询/删除/清空, DV 异常管理.

#### 与原文不一致处(⚠️ 标记说明)

- RTD 1.1 角色权限管理: 上扬原文为 "支持根据分组角色进行权限管理, 可管控到具体某一个功能 (按钮), 如系统远程发布", 赛美特 RTD 仅声明权限管理未说明细粒度, 交集裁掉了 "管控到按钮级" 的细粒度描述, 只保留两方都覆盖的角色权限管理本体.

#### 原文来源选择说明

上扬 RTD 文档以派工业务能力为主线, 描述更具体, 故角色管理, 版本管控, 在线发布, 操作履历等 bullet 均取上扬原文; 用户管理部分上扬仅一句总述, 赛美特有明确的用户新建/查询/编辑/删除 bullet, 故取赛美特原文. 两方 RTD 文档侧重点差异大 (上扬偏派工运行, 赛美特偏平台配置对象管理), 交集集中在权限, 版本发布与履历三处, 无数值指标类差异.

### AMA

#### 仅上扬独有的功能(未纳入交集)

1. POD Location: Stocker, OHB, NTB, EQP, Exchanger 等设备间全自动化流程.
2. Full Auto 场景: 基础场景与专项场景.
3. Multi Lot 约束.
4. 全局派工因素 (Critical Ratio, Move Target 等).
5. Q-time 管理与报警, Q-time Loop 监控.
6. 设备派工优先调度 (同 PPID 优先, 同组内机台优先).
7. Where Next 功能: 下货时间预判并与 MCS 联动, 上货顺序预判, Reroute 支持.
8. 最佳搬运方案: 设备到设备为被要求且首推的搬送方式.
9. Port 预约与 Queue 机制 (预约满后预约到附近 NTB/OHB).
10. MES 操作支持: Track In, Job Control, Merge, Create Monitor Wafer, Auto Hold/Release 等.
11. 规则调试, 模拟及自定义组件开发 (交集 bullet 已裁减该部分).
12. 外部数据导入工具: Flat 文件, CSV 文件或其他数据库导入.
13. 报表输出: 不派工原因, 系统性能, 执行效率报表, 定期生成并以 Text/Excel/HTML 保存到自定义目录.
14. 管理界面: 基础资料设置, 启用功能, 参数设置等界面交互.
15. 数据同步支持选定列同步.
16. 在线加列不影响系统运行.
17. 不弱于 Oracle OGG 的实时数据同步.
18. 负载均衡: 平均分配不同 Rule 的执行.
19. 缓存优先, 自动编译.
20. 系统级版本控制与回退, 事务及异常处理.

#### 仅赛美特独有的功能(未纳入交集)

1. 表单模型建模 (流程中的人机交互界面).
2. 手动发起流程 (上扬仅事件/定时/自定义触发).
3. 流程实例监控: 执行进度, 节点状态与执行结果的实时查看及全局流程实例监控.
4. 应用管理与应用中心: 应用创建, 编辑, 发布, 列表浏览与详情查看.
5. 常规后台任务的调度与管理.
6. 连接策略与连接模板的定义与复用.
7. 组织架构维护.
8. 功能菜单的细粒度配置.
9. 系统基础字典维护.
10. 系统公告发布与管理.
11. RTD 数据同步 (保证派工规则与基础数据一致).
12. 全局资源锁的查看与管理界面.
13. 系统各服务运行状态实时监控.
14. 在线用户查看与管理.
15. 前台服务: 首页导航与数据概览, 流程中心 (待办任务, 我的草稿, 流程追踪, 代理设置, 前台发起流程).

#### 与原文不一致处(⚠️ 标记说明)

- AMA 1.2 消息通知: 上扬原文仅为 "提供消息管理等系统基础管理能力", 赛美特原文为 "支持站内通知、邮箱、企业微信、短信等多种通知方式", 单方文字均不能准确表达交集, 故合并改写为 "提供消息管理与多渠道通知能力 (站内通知、邮箱、企业微信、短信等)".
- AMA 2.1 可视化开发工具: 上扬原文为 "提供快速、可视化的开发工具, 基于流程引擎开发 Rule, 支持调试、模拟及自定义组件开发; 有详细的 Log 以及相应的 Log 分析工具, 有相应的系统异常报警机制", 其中调试/模拟/自定义组件与 Log 分析, 异常报警为上扬独有, 交集仅保留两方都覆盖的 "可视化开发工具, 基于流程引擎开发 Rule" 部分.
- AMA 3.1 外部系统集成: 上扬原文列举 MES, APC, PMS, ERP, MCS, Signoff, YMS 等周边系统的直接交互, 赛美特原文为数据源, API, 消息队列等连接的配置与管理, 两方粒度与视角不同, 故合并改写为一条覆盖两方交集的集成能力描述.

#### 原文来源选择说明

上扬 AMA 文档在派工业务语义 (触发方式, 对象锁定, 在线发布) 上描述更具体, 相关 bullet 取上扬原文; 赛美特 AMA 文档在平台能力 (流程模型, 流程版本管理, 系统日志) 上定义更清晰, 相关 bullet 取赛美特原文. 两方 AMA 文档结构差异较大 (上扬偏搬送派工业务场景, 赛美特偏流程平台后台管理), 交集集中在权限, 消息, 建模, 触发, 版本发布, 集成, 锁定与日志. 两方均无数值指标类功能点, 无数值差异需标注.
