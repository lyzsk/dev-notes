## AMS

### 仅上扬独有的功能(未纳入交集)

- 用户及用户组的操作历史记录追溯
- 按角色的警报源授权 (警报数据范围隔离)
- 对用户组及用户的角色关联授权 (Authorization Mgt)
- 设备类型管理及过滤规则配置 (Equipment Type Management)
- 设备内腔体 (Chamber) 管理
- 设备信息批量导入导出, 设备可用 / 不可用状态控制
- 设备类型授权 (按角色) 与设备授权 (按用户) 及授权操作历史
- 设备告警任务配置查看 (Alarm Task Configuration)
- 警报源管理 (Alarm Source 新增 / 编辑 / 启停 / 回调等)
- 警报分类管理 (Alarm Classification)
- 告警策略管理: 系统策略与设备策略, 含批量导入导出 (Alarm Strategy / Equipment Alarm Strategy)
- 基于 FTP 文件的告警策略管理 (Alarm Ftp Config)
- 告警作业 (Alarm Task) 多条件组合查询与执行进度查看, Equipment Alarm Task 独立查询入口
- 菜单配置管理
- 告警动作设定 (Action 设定)
- 告警报表: 统计结果导出, 单日告警明细报表, 告警有效性分析, 告警完成情况统计, 跨日期告警对比分析

### 仅赛美特独有的功能(未纳入交集)

- 报警看板显示内容与刷新参数的配置
- SMTP 邮件发送参数配置与 Alarm 通知邮件模板自定义 (与上扬 "通知任务设定" 为同一通知配置能力的不同粒度, 交集仅取 "通知任务设定" 通用表述, 邮件相关具体配置未纳入)
- 系统通用参数设置
- 报警统计口径与维度的自定义配置
- Alarm Code 及描述信息维护
- 手动操作: Alarm 人工状态切换 (响应 / 关闭 / 终止), 报警转单记录, 报警作业人工执行
- 报警触发动作 (通知, 联动操作) 记录查询与追溯
- 第三方通知平台集成 (企业微信, 飞书, 短信)
- SSO 统一登录集成

### 与原文不一致处(⚠️ 标记说明)

- 无. 所有交集 bullet 均为两方原文原样引用, 未做合并或改写; 两方亦无同能力不同数值的指标项.

### 原文来源选择说明

- 结构骨架采用赛美特较简明的组织方式 (用户权限 / 设备 / 报警配置 / 查询统计 / 看板 / 系统管理), 重新连续编号.
- 具体 bullet 按 "最准确, 最具体, 最完整" 原则选取: 报警历史筛选条件, 次数统计维度, 设备维护字段, 规则维护操作等以上扬原文更具体, 取自上扬; 角色, 级别 (Severity), 数据字典, 日志, 产线结构树等以赛美特表述更完整, 取自赛美特.
- 上扬 "告警记录 / 单日明细查询" 与赛美特 "报警历史记录" 合并为一个交集点 (报警历史查询), 取上扬筛选条件更具体的原文.

## PMS

### 仅上扬独有的功能(未纳入交集)

- 基础数据管理: 自动同步 MES 设备及 Location, 建立 FAB / 区域 / 设备对应关系, 设备区域分配
- Checklist Step 内容支持中文汉字 (内容本地化)
- Checklist 填写值的 Report 功能与趋势图查看
- 机台状态关联设定: PM 各 Action (开始 / 暂停 / 取消 / 完成 / 延期) 与机台状态的联动配置
- 维修保养历史记录多条件追溯 (时间段, Scheduled / Unscheduled 分类, Work Order, PM State, Module 等)
- 与 MES 深入整合: 机台 Module, 机台状态及状态转换仅需定义一次
- PTMS 备件管理全部功能 (2.1-2.14): Parts 领入 (手工 / SAP 接口), 组装与替换, 循环寿命控制, 唯一性跟踪与履历, Lifetime 逾期报警, 替换卡控, 安全库存报警, 白名单与图片, Part Group, 属性修改与维修, 报废, Part List 查询导出, Part 详细信息查询, Return Part 退库

### 仅赛美特独有的功能(未纳入交集)

- 基于时间周期与使用情况相结合的混合类型保养
- 通过 API 供其他系统创建临时维修保养单
- 计划性维护保养任务的命名规则定义
- PM 保养管理所需的原因代码设置
- 系统默认签核逻辑 (权限群组设定) 管理保养模板与 Checklist 模板
- Checklist 步骤级共享附件 (SOP 文件或图片)
- 关联保养计划合并执行 (如周保养与月保养合并)

### 与原文不一致处(⚠️ 标记说明)

- 无. 所有交集 bullet 均为两方原文原样引用; 两方亦无同能力不同数值的指标项.
- 备注 (非改写, 仅语义对应说明): 上扬 "按照 Chamber 进行配置" 与赛美特 "按照子设备进行配置" 视为同一能力, 交集取上扬原文; 上扬 "Early Due / Due / Overdue" 与赛美特 "提前预警 / 规定时间 / 逾期时间" 对应; 上扬 "Comment" 与赛美特 "备注" 对应.

### 原文来源选择说明

- 结构骨架采用赛美特 PMS 的组织方式 (类型触发 / 模板配置 / Checklist / 排程通知 / 执行卡控 / 查询报表 / 权限), 七个二级功能均有交集.
- 两方在 PMS 部分措辞高度一致, 绝大多数 bullet 上扬原文更完整 (如 Due 窗口含 Inhibit 卡控, PM 预约含强制切换, 报警整合一条覆盖 Early / Due / OverDue 三点), 故主要取自上扬.
- "任务操作" 取赛美特原文 (开始 / 中止 / 取消 / 继续 / 完成 / 延迟), 因其恰好等于两方操作集合的交集, 上扬原文含重新计划, 激活, Adhoc 等赛美特未列出的操作.
