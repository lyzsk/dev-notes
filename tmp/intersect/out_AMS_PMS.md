## AMS Function List

### 1. 用户与权限管理

#### 1.1 用户与用户组管理

支持系统用户账号与用户组的统一管理.

- **账号管理**: 支持用户账号的创建、维护与停用.
- **用户组维护**: 支持用户组的新增与查询, 便于按部门或职能进行群组化管理.

#### 1.2 角色与权限管理

支持基于角色的权限体系构建与功能访问控制.

- **角色管理**: 支持系统角色的创建与维护.
- **菜单授权**: 支持按角色进行菜单授权, 控制功能访问范围.

### 2. 设备信息管理

#### 2.1 产线结构与区域管理

支持工厂下产线层级结构与区域的维护.

- **产线结构树**: 支持 Factory / System / Tool 等产线结构树信息的维护.

#### 2.2 设备基础信息维护

支持设备基础信息的全生命周期维护.

- **设备维护**: 支持设备的新增 (关联工厂 / 区域 / 类型, 配置过滤规则), 编辑 (除名称外均可修改) 及删除.

### 3. 报警配置管理

#### 3.1 报警级别管理

支持警报等级的自定义管理.

- **报警级别管理**: 支持 Alarm 严重等级 (Severity) 的定义与管理.

#### 3.2 报警规则管理

支持告警规则的定义与维护.

- **规则维护**: 支持告警规则的新增, 编辑 (字段卡控与新增逻辑一致), 删除及启用 / 禁用.

### 4. 报警查询与统计

#### 4.1 报警历史查询

支持告警历史记录的多维度检索.

- **多维筛选**: 可根据起始日期, 区域, 设备类型, 设备名称, Alarm ID/Code 等条件筛选设备告警历史记录.

#### 4.2 报警次数统计

支持告警发生次数的多维度统计.

- **次数统计**: 支持按 "设备+Alarm ID" 等维度统计告警发生次数.

### 5. 报警看板

#### 5.1 报警看板

支持告警态势的一站式可视化监控.

- **工作台视图**: 展示各种告警数据量, 告警分布情况, 处理率等关键监控指标.

### 6. 系统管理

#### 6.1 数据字典

支持系统基础数据字典的维护.

- **数据字典**: 支持系统数据字典的定义与维护.

#### 6.2 操作日志

支持系统操作日志的记录与审计追溯.

- **日志查询**: 支持系统日志的检索与查询, 便于问题排查与审计追溯.

#### 6.3 告警通知设定

支持告警通知任务的设定.

- **通知任务设定**: 支持通知任务的设定.

## PMS Function List

### 1. 机台保养 (PM Management)

#### 1.1 保养类型与触发机制

支持计划性与非计划性, 多触发类型的维修保养管理.

- **计划性保养**: 支持计划的维修保养管理.
- **非周期维修**: 支持非周期的维修管理.
- **时间触发**: 支持基于时间的周期性保养管理.
- **使用触发**: 支持基于使用情况的周期性保养管理, 与 EAP 整合, 接收对应参数值进行触发 (例如 Wafer Count 及 RFTM).

#### 1.2 PM 模板与计划配置

支持标准化的保养模板配置与精细化的排程规则设定.

- **模板管理**: 支持增加、修改、复制维修保养管理模板, 计划性维护保养模板支持版本控制.
- **多维度配置**: 计划性维护支持按照设备、按照 Chamber 进行配置.
- **Checklist关联**: 支持选择所需的已建立 Checklist 模板.
- **Trigger参数**: 支持设置 PM Trigger 所需的递增性及替换性参数.
- **Due窗口管理**: 支持设置 Early Due, Due, Overdue 时间, 并可定义 Overdue 时是否自动 Inhibit 机台.
- **排程类型**: 支持定义浮动型及静态型 PM, 即根据 PM 完成时间或固定的时间来重新规划下一次 PM.
- **附件与部品**: 支持添加 PM 过程中用户需要的共享附件 (例如 SOP 文件);支持定义 PM 需使用的部品详细信息 (料号与描述信息).
- **预估时间**: 支持设置 PM 保养所需的预估保养时间.

#### 1.3 PM Checklist 管理

支持结构化的检查步骤定义, 参数卡控与模板复用.

- **步骤与模板管理**: 支持添加、删除、修改 Checklist Step, 以及增加、修改、复制维修 PM Checklist 模板.
- **导入导出**: 支持 Export 及 Load Checklist 模板.
- **参数类型与提示**: 支持设置必填及选填参数;未填写必输入参数时, Complete 会进行报警提醒.
- **规格卡控**: 支持规格限制及 Check, 可定义超出规格时是否需要填写 Comment.
- **复机检查**: 支持设置 PM 复机 Checklist 的检查执行步骤.
- **故障单关联**: 支持 UnScheduling Down 时手动建立故障单 Work Request, 并手动添加所需 Checklist.

#### 1.4 保养排程与预警通知

支持智能化的排程辅助, 强制管控机制与主动式报警通知.

- **时间预测**: 支持预测维修保养的时间.
- **自动排程**: 周期性保养完成后自动安排下一个维修保养计划.
- **Overdue强制切换**: 达到强制维修保养条件 Overdue 时, 强制切换设备 /Chamber 可用性.
- **PM预约**: 根据用户 UI 设定预约 PM, 设备到达预约 PM 时间时强制切换设备 /Chamber 可用性.
- **AMS报警整合**: 与 AMS 系统整合, 在用户定义的 Early, Due, OverDue 时间点将报警发送给 Alarm 系统.

#### 1.5 保养执行与关键配件卡控

支持完整的 PM 作业流程控制与 Key Parts 更换的强制卡控.

- **任务操作**: 维修保养任务支持开始、中止、取消、继续、完成、延迟等操作.
- **Key Parts定义**: 定义 PM 计划时可定义需更换的 Key Parts List (料号与数量).
- **更换卡控**: 完成 PM 时检查需更换的 Key Parts 是否已完成更换, 否则 PM 不可以结束;更换可设置是否强制, 且需要填写 Comment.

#### 1.6 查询与报表

支持多维度的维保计划与状态查询.

- **灵活查询**: 可按照设备、拟定的设备群、默认时间范围进行查询.
- **计划与状态**: 支持查询维修保养管理计划及维修保养管理状态.
- **清单查看与导出**: 支持查看维修保养清单列表, 并支持 Export 导出.

#### 1.7 权限管理

支持统一的用户权限管理机制及与 MES 的整合.

- **权限机制**: 提供用户权限的管理机制.
- **MES 整合**: 权限管理模块与 MES 进行整合, 减少用户重复定义.
