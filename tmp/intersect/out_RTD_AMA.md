## RTD Function List

### 1. 用户与权限管理

#### 1.1 角色与权限管理

支持基于角色的用户与权限管理, 实现精细化访问控制.

- **角色管理**: 支持对所有人员按照分组角色进行管理.
- **角色权限管理**: 支持根据分组角色进行权限管理. ⚠️

#### 1.2 用户管理

支持用户账号的新建, 查询与维护.

- **用户新建与查询**: 支持新建用户及查询用户.
- **用户维护**: 支持编辑用户及删除用户.

### 2. 规则版本管理与发布

#### 2.1 版本与发布管理

支持规则与配置对象的版本管控及在线发布, 保障系统持续演进.

- **版本管控**: 可保存多个版本, 可随时进行版本切换及查询.
- **在线发布**: 支持在线功能发布, 并支持在下次运行时自动调用系统发布的最新版本规则.

### 3. 履历管理

#### 3.1 操作履历

支持操作历史的完整记录与追溯.

- **操作履历**: 提供 RTD 操作历史记录, 支持履历追溯.

## AMA Function List

### 1. 系统管理

#### 1.1 角色与权限管理

支持角色, 用户与权限管理, 按角色进行访问控制.

- **角色与权限管理**: 提供角色管理、用户管理、权限管理等功能, 按角色进行访问控制.

#### 1.2 消息通知

支持系统消息与多渠道通知能力.

- **消息通知**: 提供消息管理与多渠道通知能力 (站内通知、邮箱、企业微信、短信等). ⚠️

### 2. 业务建模与规则触发

#### 2.1 可视化建模

支持自动化派工流程与规则的可视化建模与开发.

- **流程模型**: 支持业务流程模型的设计与维护, 定义自动化派工 Workflow.
- **可视化开发工具**: 提供快速、可视化的开发工具, 基于流程引擎开发 Rule. ⚠️

#### 2.2 触发方式

支持多种触发方式驱动 Dispatching Rule, 确保派工响应的实时性与灵活性.

- **事件触发**: 当 Load Port Ready to Unload Event 报出时, 自动 Trigger Dispatching Rule.
- **定时触发**: 基于 Periodic Watchdog 周期性 Trigger Dispatching Rules.
- **自定义触发**: 基于设备状态及其他用户自定义的客制化状态 Trigger Dispatching Rules.

#### 2.3 版本管理与发布

支持流程与规则的多版本管理与在线发布, 保障生产流程平稳迭代.

- **版本发布与管理**: 支持流程版本的发布、切换与历史版本管理, 确保生产流程平稳迭代.
- **在线发布**: 支持在线功能发布, 并支持在下次运行时自动调用系统发布的最新版本规则.

### 3. 系统集成与运维

#### 3.1 外部系统集成

支持与外部系统及数据源的连接与实时信息交互.

- **外部系统集成**: 支持与 MES, APC, PMS 等周边系统以及数据源, API 等外部连接的直接交互与集成配置. ⚠️

#### 3.2 对象锁定

支持对象锁定功能, 避免资源争抢, 提高派工效率.

- **对象锁定**: 支持对象 (Lot/Port/POD 等) 锁定功能, 避免多个设备或 Port 争抢同一个 Lot, 以便提高派工效率.

#### 3.3 系统日志

支持系统运行日志的记录, 查询与追溯.

- **系统日志**: 支持系统运行日志的查询与追溯.
