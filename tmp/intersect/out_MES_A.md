## MES Function List (A. 建模与基础)

### 1. 工厂基础建模

#### 1.1 厂区与设备对应

支持工厂区域层级与设备的对应关系建模.

- **区域与设备对应**: 支持 FAB/ 区域 / 设备对应关系定义, 设备可分配到不同区域, 用于工艺工程师 / 设备工程师 / 制造等不同目的.

#### 1.2 代码、编号规则与属性扩展

支持业务代码、对象编号规则及模型属性扩展的定义.

- **原因代码**: 灵活配置报废、扣留、释放、终止等各类原因代码, 并方便查询、展示.
- **编号规则**: 基于业务需要灵活配置产品批次 (子母批)、工艺配方、产品、制造资源等对象的编号规则.
- **自定义属性扩展**: 支持对工厂、机台、机台组、产品、产品组、工艺流程、层、段、站点、EDC, Recipe, Reticle、探针卡、Q-Time、批次、晶圆等所有模型对象自定义扩展属性.

#### 1.3 版本控制与历史记录

支持建模对象的版本管控、历史追溯与权限分离.

- **版本控制**: 产品、工艺流程、层、段、站点、Logical Recipe, Physical Recipe, EDC, Reticle、探针卡、Q-Time 等模型对象支持版本控制, 每个对象仅唯一 Active 版本, 新版本生效后旧版本冻结不可随意激活, 版本号自动增加, 所有变更可记录追踪.
- **版本与历史**: Process, Product, Subplan 要具备版本控制功能, 都要具备历史查询详细的记录.
- **设置与查看权限分离**: 关键模型参数的设置功能与查看功能分开, 避免查看人员拥有设置权限.

### 2. 产品建模

#### 2.1 产品定义与流程关联

支持产品及其属性的定义、产品组管理与产品流程共用.

- **产品定义**: 支持定义工厂生产的成品 / 半成品等产品及其属性, 可定义产品的 Wafer 类型, 产品责任人.
- **产品组**: 支持产品组 (产品的 Type 定义) 的定义. ⚠️
- **共用 Flow**: 不同 Prod 可以共用同一个 Flow, Prod 可以设定基本属性.

### 3. 工艺流程建模

#### 3.1 流程架构与资源关联

支持多层级、模块化的流程架构及工艺资源的关联配置.

- **多层级模块化建模**: 支持工艺流程、工段、站点多层级, 支持模块化建模, 各级模型对象可复用和共享, 多个产品可共用相同工艺流程.
- **资源关联**: 支持基于产品、工艺流程、工序关联机台组、机台、Recipe, EDC, Q-Time、光刻板、探针卡.
- **按产品配置**: Recipe, EDC, Q-Time 均可按产品单独配置.
- **关联使用**: 支持 EDC Plan 与产品、工艺流程、站点关联使用.

#### 3.2 多路径流程与返工

支持分支跳转、返工路径与机台绑定卡控.

- **分支跳转**: 允许手动或在预设条件下灵活跳转到正常产品流上的其他分支流程.
- **返工路径**: 支持主工艺路径工序绑定计划返工路径 (Rework), 可定义返工站点和指示, 返工制程可被多个主制程调用.
- **机台绑定**: 支持不同 Stage, Layer 卡控必须使用相同的机台或者机台组, 前量和后量卡控同一台设备.

#### 3.3 流程查看、比较与导入导出

支持工艺配置的层次化查看、版本比对与批量导入导出.

- **层次结构查看**: Process Specification Editor 是一个类似于 Windows-Explorer 风格的查看器, 提供带有 Process Planning 信息的层次结构.
- **流程比较**: 支持比较两条工艺流程 (不同流程或不同版本) 并标记不同之处.
- **批量导入导出**: 导入 / 导出格式统一, 导出数据修改后可直接导入; 功能导入及 Excel 模版导入两种方式均需支持比较与校验防错.

### 4. Recipe 建模与管理

#### 4.1 Recipe 定义与维护

支持设备配方的定义与生命周期维护.

- **配方维护界面**: 提供修改、复制、查询、状态修改等维护功能, 支持配方启用和停用.
- **多腔设备支持**: 支持多腔设备, 支持腔体串并联.

#### 4.2 多腔配方管控

支持依据设备与腔室状态的配方选择管控.

- **配方选择**: 相应的 Recipe 由设备和腔室状态组合定义, 当执行该过程时, 系统会选择最佳的可运行配方 (RCP 系统管控).

### 5. Q-Time

#### 5.1 Q-Time 定义与关联

支持 Q-Time 的时间、动作设定与复用关联.

- **Q-Time 设置与动作**: 提供最小和最大 Q-Time, 多步之间的 Q-Time, 并提供不同的触发设置, 当系统超过 Q-Time 时自动触发, 不同的时间触发不同的警告.
- **复用关联**: 定义好的 Q-Time 可复用关联到不同产品、工艺流程、工序.

#### 5.2 计时控制与超时处置

支持 Q-Time 的多点计时、继承恢复与超时处理.

- **嵌套与触发**: 支持 Q-Time 嵌套, 交叉等设定, 支持 TrackOut 触发, TrackIn 触发, TrackIn 结束, TrackOut 结束; Lot 支持多个 Q-Time.
- **CancelTrackIn 恢复**: 具备 CancelTrackIn 后 Q-Time 恢复计时的能力, 当 Lot 处在 Q-Time 结束站点时, 如果发生 Cancel TrackIn, Q-Time 需要继承回来.
- **超时扣留**: 加工过程中 Q-Time 超时时, 可在出站后触发批次扣留动作.
- **分批继承**: 分批时, 子批要能自动继承母批的 Q-Time.

### 6. 机台建模

#### 6.1 设备分级、属性与设备组

支持主/子设备分级建模、属性能力定义与设备组管理.

- **设备分级建模**: 支持主机台与子机台 (Chamber, Tube, Loadport 等) 分级定义及关联; 支持设备分类管理及类型 / 型号定义, 含倒片机 (Sorter)、光刻联机设备 (Inline) 等特殊设备.
- **设备属性定义**: 支持定义主、子机台属性, 含污染等级、状态切换模型、加工类型、最大加工能力、单次并行加工能力 (Batch) 等, 及机台可接受的载具类型.
- **批量数量定义**: 支持设备批量数量的定义, 例如最大和最小 Wafer 数, 最大和最小载体数量的定义.
- **设备能力定义**: 支持 Offline, Online Local, Online Remote 等设备能力的定义.
- **机台组关联**: 支持通过机台 / 机台组关联产品、工艺流程、工序, 工序增减机台时按机台组快速变更, 无需对所有产品工艺流程逐一更新.
- **腔室模式**: 可以定义腔室 ID, 可以定义串行 / 并行 / 组合三种模式.

#### 6.2 设备状态定义与转换

支持设备状态体系、转换规则与权限管控.

- **状态模型与转换**: 支持主、子设备状态及状态转换配置, 标准支持 E10 及用户自定义状态; 支持事件发生时实时自动触发状态切换 (如批次入机台时机台由闲置转运行), 也支持手动状态转换.
- **状态转换权限**: 设备状态转换支持按角色权限设定, 无权限则不能转到对应状态; 权限配置支持批量导入导出及变更调整.
- **状态联动**: 腔室状态与批次进出站联动, 腔室状态将对应转换为不同的设备状态; 主设备状态将根据腔室状态合理的自动更改.

#### 6.3 机台限制 (Tool Constraint / ECS)

支持按主/子机台设定多维限制条件与动态管控.

- **控制条件**: 可根据主机台和子机台 (Tube, Chamber 等) 设置产品 / 工艺流程 / 段 / 站点 / 配方 / 光罩 / 探针卡 / 晶圆片数量 / 批次等控制条件, 支持通配符, 可单独或组合设定, 同一主 / 子机台允许设定多条.
- **正反向设定**: 支持机台限制正反向设定.
- **动态条件管控**: 支持批次在特定时间区间内限制最大可运行晶圆片数或批次数.

#### 6.4 设备查询与历史

支持设备信息查询与加工历史追溯.

- **设备列表查询**: 可查询全部设备列表, 含设备编码、名称、主 / 子状态、旧主 / 子状态、维修信息、设备组、机台责任人、最新动作、备注等.
- **历史记录**: 记录机台所有过程详细信息, 含机台信息、状态转换信息、批次加工信息、上下光刻板信息.

### 7. 特殊设备与作业管理

#### 7.1 炉管 (Furnace)

支持炉管组批、动态配方与监控片数据共享.

- **组批模式**: 支持生产批与炉管监控批一起组批, 也支持只有生产批或监控批单独组批的跑货模式.
- **动态 Recipe 调整**: 通过 ARG 系统针对 BatchLot 的 Wafercount 区间, 进行 Layout Recipe 动态调整.
- **量测同步**: 炉管 MonitorLot 做完量测后, 可以对 Batch 内的生产批次进行同步的量测过帐以及量测数据贴值.
- **挡片填充**: 支持炉管机台填充挡片 (Dummy) 机制.

#### 7.2 WET 批量作业

支持 WET 设备的单批/双批批量跑货.

- **载具组合**: 可选择同时跑 2 个载具或 1 个载具.

#### 7.3 倒片机 (Sorter)

支持 Sorter 的流程内/非流程作业与任务类型管理.

- **作业模式**: Sorter 机台要能支持流程内 (Inline Sorter) 和非流程 (Adhoc Sorter) 的作业; 支持 Inline Sorter Step, 能够按照用户的设定完成 Sorter 动作, 例如 Split, Flip, ReadT7Code 等.
- **SorterJob 类型**: 支持 SorterJob 基本类型, 如 Exchange, Split, Merge, WaferFlip, ReadT7Code.

#### 7.4 载具与污染等级卡控

支持载具与批次污染等级的开工校验及机台污染等级管控.

- **开工校验**: 校验载具、批次污染等级, 通过才能开工.
- **污染等级卡控**: 机台设定可接受的 Lot 的污染等级, 支持设定多个污染等级, 污染等级需要精确匹配.
