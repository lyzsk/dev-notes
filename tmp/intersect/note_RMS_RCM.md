### RMS

#### 仅上扬独有的功能(未纳入交集)
- 比对方式细化: CheckSum, Recipe 修改时间比对, Recipe Binary Body 分析与解析, 字符串/整数/小数数据类型
- 比对呈现: 可选择显示全部内容或仅显示差异
- Recipe 导入导出功能及权限管控
- 批量加载可选择每次加载数目
- 各项排序
- EC 之外的状态变量 (SV) 管控, 按设备或按 Recipe 设定 EC/SV 管控范围
- 系统内部签核, 可设定各区域模块签核人/组
- 签核可视性: 签核人员可查看关键参数内容
- 批量签核的特殊标注记录
- 系统管理: 用户组与权限分级管控, 用户子管理员, 密码加密与重置, 会话管理 (登入登出记录, 自动登出), 日志保存时间配置
- 高可用架构: RMS Server NLB Failover
- 数据库兼容: Oracle, PostSql 等

#### 仅赛美特独有的功能(未纳入交集)
- Equipment Category: 设备程序类型分类设定
- Equipment List: 设备关联角色/负责人员及组别设定, 通知测试邮件发送
- 从设备端获取程序列表
- 版本状态流转细化: Set Frozen, Deactivate 等操作
- 参数规范 Excel 导入
- Recipe Template 管理: 按设备类型查看模板, 公式编辑, 模板删除
- 待审批查询: 按区域/设备类型/设备查看待 Approve 程序
- Recipe Status: 程序状态查看与关键字模糊查询, Delete 程序
- Golden Recipe 机制细化: 基准设备设定 (Allow Share Recipe, Auto Share 勾选), 共享关系查看与修改, 非基准设备独立 Active 版本
- Strict Parameter: 严格检查参数锁定, 不允许 Uncheck
- Key Param Spec 自动套用 (Load Recipe 时)
- Uncheck Param List: 免检查参数机型化配置与自动 Uncheck
- Recipe Parse Adaptor 不停机上线/更新
- Offline Audit: 模拟 EAP Validation 比较与批量 Offline Audit
- 机台校验结果汇总报表
- EC 比对: 设备常量与设备中数值比较, 同类型设备间常量差异比较

#### 与原文不一致处(⚠️ 标记说明)
- 1.1 多种比对方式: 上扬原文含 CheckSum, 修改时间, Binary Body 等独有方式, 赛美特仅细分 FullBody 与 Parameter (Absolute/ByRange/ByTolerance/ByList), 交集仅保留双方共同覆盖的 FullBody 与 Parameter 两种形式, 故改写.
- 1.1 Spec 模板复用: 合并上扬"多设备共用模板设定 Recipe Parameter Spec"与赛美特"通过模板复制参数规范, 参数规范可保存成模板"的共同部分; 上扬"不同 Step 相同参数批量设定 Spec"与赛美特"Excel 导入"为各自独有, 未纳入.
- 1.1 关键参数管理: 上扬为"设置关键参数并高亮显示", 赛美特为"根据机型设置 Key Param 的 Spec 范围", 表述与侧重点不同, 取双方共同覆盖的"关键参数设定"能力合并改写, 各自细化在括号中注明.
- 1.2 上传下载与批量作业: 上扬将上传/下载/Spec 批量备份/批量共享合为一条, 赛美特拆分为上传, Download, Batch Load, Batch Set Active 多条, 合并改写为双方共同覆盖的上传, 下载及批量操作 (含批量 Active).
- 1.2 Recipe 共享: 上扬为"Recipe 批量共享", 赛美特为 Golden Recipe 基准设备自动共享机制, 实现机制不同, 取共同能力"同型号设备间共享 Recipe"改写.
- 1.3 变更履历查询: 上扬为"版本变更历史详情"一条带过 (同条另含独有的"各项排序"), 赛美特拆分为 Standard/Sequence 多条且含子程序比较开关履历等细化, 合并改写为双方共同覆盖的 Recipe/Sequence 版本变更履历及 SPEC 修改履历.
- 2.1 多设备共享: 上扬为"多设备之间共享 EC/SV 设定" (含独有的 SV), 赛美特为"同类型设备之间的设备常量设定 Copy", 取 EC 部分合并改写.
- 3.1 批量签核: 截取上扬"支持批量签核" (其"特殊标注记录"为独有细节), 赛美特对应能力为批量 Set Active 时产生一张签核审批单.

#### 原文来源选择说明
- 管控列表与 ByPass, 校验异常处理, 多版本管控等条目上扬描述更完整且恰好覆盖双方交集, 取上扬原样.
- 设备间/版本比对, 通知时机, 审批操作, EC 常量定义与履历等条目赛美特表述更具体准确, 取赛美特原样.
- 两方粒度差异较大或各含独有细节的条目 (比对方式, 模板, 关键参数, 上传下载批量, 共享, 变更履历, EC 共享, 批量签核) 采用合并改写并加 ⚠️ 标记.

### RCM

#### 仅上扬独有的功能(未纳入交集)
- 集中管控架构: 1 套 RCM 控制所有设备, 防单点失效
- 多语言支持 (英语/中文)
- 网络协议 TCP/UDP
- PC 端 KVM 与触摸屏画面联动
- 用户登录历史与远程控制历史记录
- RCM 用户配置管理 (添加/删除/编辑/查找)
- 部署要求与双机热备
- 核心交换机通讯速率规格 (4320 Mpps/36000 Mpps)
- 硬件通讯异常报警 (KVM 断开弹窗)
- 客制化能力
- 对接本司账号系统并拉取部门结构, 按部门/人员设定权限
- 接入 RCM 硬件数目无限制, 无额外费用
- 与 EAP, MES, PLC, RMS 等系统的自定义通讯接口
- 断连状态显示与报警提示
- RPA 融合
- 设备端物理按钮近远端切换
- 本地警示文本与本地操作失效
- 声光报警 (报警灯/蜂鸣器)
- EQ Monitoring 状态实时同步
- 三级分级权限 (Operate/工程师/管理者) 与分组管理
- 设备报警时远程画面外框颜色变化提示
- 实时性指标: 操作网络延时 < 0.1 s, 画面延时 < 65 ms, 无闪烁无残留
