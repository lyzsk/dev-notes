# 基础FLOW建模
- 所有基础资料的维护都需要保留操作历史记录，并支持按时间、对象等条件查询。
- 支持查询所有建模数据（如工厂建模、产品、工艺流程、配方、工程数据收集、设备、代码等）并导出。
- 支持快捷查询各对象被谁使用。
- 可以自定义扩展所有模型对象的属性，模型对象包含不限于工厂、机台、机台组、产品、产品组、工艺流程、层、段、站点、工程数据收集、配方（Recipe）、光罩（Reticle）、探针卡、容许时间（Qtime）、批次、晶圆。
- 在所有数据生效前，必须检查所有设定彼此间是否一致，数据如有变更则需与当前实际情况检查是否一致，是否可以变更。
- 所有模型对象支持版本控制，模型对象包含但不限于产品、工艺流程、层、段、站点、逻辑配方（Logical Recipe）、物理配方（Physical Recipe）、工程数据收集、光罩（Reticle）、探针卡、容许时间（Qtime）等。版本变更时遵循如下要求：
- 版本的控制要能支持Active版本的定义，每个对象只有唯一的激活（Active）版本号。工程师可以从多个版本中指定激活（Active）版本；
- (新版本生效后，旧版本将会被冻结，工程师不可以随意指定激活（Active）版本；
- 所有的改变都能够被记录下来并且可以被追踪；
- 涉及版本变更时，版本号能够自动增加。
- 关键模型参数，设置功能和查看功能需要分开，避免查看人员有设置权限。
- MES系统模型逻辑和用户需求逻辑出现冲突时，应以用户需求为准进行调整。
- 修改工艺量测参数采样计划时，按MES内嵌签审流程进行工艺量测参数的重设目标值，无需升版。
# 厂区建模管理
- 支持多工厂建模及生产，8英寸及先进封装工厂、微组装工厂，每个工厂独立，批次产品、流程、批次隔离，且支持后续新工厂模型扩展建立和生产流片运转。
- 支持工厂（Fab）、区域（Area）、片区（Bay）、暂存位（Bank）等定义。
- 支持扩散（DIFF）、光刻（Litho）等工作区域定义。
- 支持工厂、区域、设备对应关系，设备可以分配到不同的区域用于工艺工程师、设备工程师、生产制造。
- 支持工厂日历。
- 支持生产排班，批次加工历史能正确记录该班次信息。
- 支持产品在多工厂切换加工。
# 机台建模
- 支持设备对象分级定义，比如主机台、子机台（chamber、Tube、Loadport等），主机台与子机台关联。
- 支持设备分类管理；同时应可定义设备类型/型号。主设备除普通设备外还有特殊设备，比如倒片机（Sorter）、光刻的联机设备（Inline）等。
- 支持定义设备对象（主、子机台）属性：污染等级，状态切换模型，加工类型，最大加工能力，单次并行加工能力（Batch）等。
- 支持定义机台可接受的载具类型。
- 支持可配置主、子设备状态及状态转换，状态转换标准支持E10以及用户自定义的设备状态。
- 支持管理设备状态模型：设备状体定义及状态切换表，支持事件发生时实时自动触发设备状态切换，比如批次入机台时机台/子机台由“闲置”转为“运行”状态等；也支持手动用MES功能进行状态转换。
- 设备状态转换支持用户权限设定，没有权限的则不能转到该种状态，比如某类角色能从某状态转换到另外一种状态。设备状态转换权限配置支持批量导入导出，支持变更调整。
- 支持配置运行时设备（主、子机台）状态检查。
- 支持手动和自动化（Online-Remote状态，通过EAP控制机台加工）操作管控。
- 支持通过机台/机台组关联产品、工艺流程、工序，各工艺流程相关工序需要增减机台时可快速变更，无需对所有产品工艺流程一一更新，比如按照机台组内新增机台即可。
- 支持批量处理，支持操作者一次处理多条记录。
- 支持设备各级对象创建、修改、复制，支持通过复制其他来生成新设备，支持批量处理。
- 数据清除及存档：提供完整功能给终端用户以及IT进行设备主数据及历史数据进行删除以及存档动作，该动作不能影响报表数据库，不能影响现有批次流片及查询。
- 可查询全部设备列表，包含设备编码、设备名称、主状态、子状态、旧主状态、旧子状态、维修信息、设备组、机台责任人、最新动作、备注等。
- 历史记录机台所有过程详细信息，包含机台信息、状态转换信息、批次加工信息、上下光刻板信息。
# 产品建模
- 支持定义产品模型，属性包含产品类型、产品负责人、DPW（每片晶圆Die数）、ERP料号、单位等。
- 源产品信息需要包关键参数、厂家、供应商原批次号。
- 支持自定义产品编码规则。
- 支持不同产品类型，如量产、科研、监控片、挡片。
- 支持产品版本管控，升版本时进行新旧版本差异比较，以确认修改正确性。
- 支持定义产品与工艺的关系，持多对一对应关系。
- 支持定义产品与原单晶产品关系。
- 支持定义产品、工艺流程、配方、机台对应关系。
- 支持产品状态定义及转换。
- 支持产品组定义。
- 支持产品创建、修改、复制，支持通过复制其他来生成新产品，支持批量处理。
- 数据清除及存档：提供完整功能给终端用户以及IT进行产品主数据及历史数据进行删除以及存档动作，该动作不能影响报表数据库，不能影响现有批次流片及查询。
- 记录历史详情。
# 工艺流程建模
- 支持工艺流程、工段、站点多层级，支持模块化建模，各级模型对象能复用和共享。
- 支持定义工艺流程各级对象类型（量产、工程、实验、返工、监控等）。
- 多个产品可以共用相同的工艺流程。
- 支持对工艺流程各级对象编号、状态、版本等管控。
- 可由工艺流程/站点来定义段/层。
- 支持基于产品、工艺流程、工序等关联机台组、机台、配方、工程数据收集、容许时间（QTime）、光刻板、探针卡。
- 主工艺流程可有多个子路径组成，包含普通路径、分支路径（Multi-Path）、返工路径（Rework）、条件判断分支等。
- 支持主工艺路径的工序上绑定计划返工路径（Rework）。
- 能够定义返工站点和指示，返工制程可以被多个主制程调用。
- 支持对工艺站点（Process steps）与量测站点（Metrology step）、前量与厚量的关系对应。
- 支持虚拟站点。
- 支持批量导入/导出，导入/导出格式统一，能够将导出的数据修改后直接导入，导入时有比较与校验防错功能。
- 支持在工段特定工序的先后顺序校验。
- 支持工艺制程信息变更签审后生效。
- 工艺流程升版本需要继承旧版工艺流程上设定的容许时间（Qtime）、采样、计划扣留等，如果由于增删站点/工段变化导致容许时间（Qtime）、实验分片单(SRC)、故障处理单(RRC)、先行批（Pilot）等起止站点失效需要识别影响并告知用户，并按合适方式进行处理，比如对在制批次处理：当新版本的流程激活（Active）时，除了特别要求的批次外，批次都应该自动升版至激活版本；作业中批次批次不能变更，等加工完成后判定非特别要求的批次外自动升版，特别要求的批次保留原版本；
- 升版时不能影响在实验分片单(SRC)、故障恢复单(RRC)、返工、超规计划（OCAP）处理中的批次，且这些批次能够在结束上述非常规工艺返回主流程后自动升版到激活的版本，当（子）工艺流程升版本导致返回点不存在时，则回到旧版的返回点并扣留批次，支持人工升版本到新版（子）工艺流程；
- 流程升版应该继承原有的容许时间/取样计划/站点扣留等设定；批次升版也要保留已经触发的容许时间；
- 对于未自动升版成功的批次，要有提示并将未成功升版的批次清单信息发邮件给相关工程师。
- 支持工艺流程各级对象创建、修改、复制，支持通过复制其他来生成新工艺流程、工序等，支持批量处理。
- 支持比较两条工艺流程（不同工艺流程或工艺流程的不同版本）标记不同之处。
- 配方（Recipe）、工程数据收集、容许时间（Q-Time）都可以按产品单独配置。
- 数据清除及存档：提供完整功能给终端用户以及IT进行工艺流程主数据及历史数据进行删除以及存档动作，该动作不能影响报表数据库，不能影响现有批次流片及查询。
- 记录历史详情
# 配方建模管理
- 流程的站点上定义的是配方，跑货时下给设备的是 设备工艺流程程序ID (PPID ,Process Program ID)，配方和 设备工艺流程程序ID(PPID)之间有一定的对应关系。
- 支持配方的分组。
- 产线包含普通独立机台、多腔（Chamber）机台、炉管机台、研磨机台、联线机台匀胶曝光显影光刻机台。配方需要支持这些机台类型的配方配置。
- 需要有维护配方的界面，包括修改，复制，查询，状态修改等。
- 更新配方，无需对所有产品工艺流程一一更新，需要仅作少量调整即可。
- 增加机台，无需对所有产品工艺流程一一更新，按照机台组内新增机台即可；并且具备查询机台组对应使用工艺流程。
- 根据配方查询哪些产品在使用。
- 记录历史详情。
- 多腔机台在派工时，需要考虑腔的状态来选择 设备工艺流程程序ID(PPID)。
- 炉管机台会根据配方和生产晶圆片的片数来选择对应的设备工艺流程程序ID(PPID)。
- 在生产批次的履历中要记录设备加工所使用的配方/设备工艺流程程序ID(PPID)。
- 支持配方的启用和停用。
# 工程数据收集计划(EDC PLan)
- 支持定义工程数据收集计划(EDC Plan)的对象：采集点、采集计划、采集参数（EDC参数）、采样规则、上下限规则、上下文内容，列出要收集的所有的需要的数据。采集计划、采集参数、采样规则、上下限规则可以复用。
- 支持工程数据收集计划(EDC Plan)各对象的版本控制及修订追踪。
- 工程数据收集计划(EDC Plan)需支持版本控制。
- 支持工程数据收集计划(EDC Plan)与产品、工艺流程、站点关联使用。
- 工程数据收集计划(EDC Plan)规格可定义数据收集的类型：Lot/Wafer 。
- 支持在采集点关联设置采集计划、采集参数、采样规则、上下限规则。
- 支持在采集计划关联设置采集参数、采样规则、上线规则。
- 支持数据采集时伴随收集的上下文信息。
- 工程数据收集计划(EDC Plan)的参数支持传入SPC，支持接受SPC结果发起一些动作，例如扣留批次（Hold lot）、锁定机台（Hold Tool）等。 对于违反工程数据收集计划(EDC Plan)规格导致OOC的时候，可以扣留批次（Hold lot）、锁定机台（Hold Tool）。
- 对于炉管设备，需支持炉管监控批次的量测数据共享给生产批次。
- 基于生产批次采集的工艺参数，如果母批前测完成后进行分批，子批应该支持前测量测结果，和支持子批进行后测及对应关系的计算。
- 支持批量处理，支持操作者一次处理多条记录。
- 支持工艺制程信息变更签审后生效。
- 支持工程数据收集计划(EDC Plan)各对象创建、修改、复制，支持通过复制其他来生成新工程数据收集计划(EDC Plan)对象，支持批量处理。
- 数据清除及存档：提供完整功能给终端用户以及IT进行工程数据收集计划(EDC Plan)主数据及历史数据进行删除以及存档动作，该动作不能影响报表数据库，不能影响现有批次流片及查询。
- 与SPC集成支持将工程数据收集计划(EDC Plan)的内容送SPC用来定义SPC 管制计划收集数据的过滤条件，如：按设备，按子设备，按配方， 按产品， 按流程；
- 与SPC集成接受SPC OOC触发的超规动作计划(OCAP)流程；
- 支持SPC 管制计划和MES的工程数据收集计划(EDC Plan)规格设置根据规则关联查询， 并支持批量工程数据收集计划(EDC Plan)规格与关联SPC 管制计划同时新增和修改；
- 支持历史详情记录。
# 选片规则
- 工程数据收集计划(EDC Plan)采集的规则可以定义，规则号可以自行编辑，可以根据不同片数的批次选择不同的片数，且wafer的位置均可以配置。
- 可忽略特殊位置。
- 从目标插槽开始搜索，从目标插槽的位置到结束插槽中所搜索到的第一片晶圆。
- 从目标插槽开始搜索，从目标插槽的位置到起始插槽中所搜索到的第一片晶圆。
- 从目标插槽开始搜索，从目标插槽的位置到结束插槽若存在晶圆，则选取第一片搜索到的晶圆，若不存在晶圆，则从目标插槽的位置到起始插槽搜索第一片存在的晶圆。
- 从目标插槽开始搜索，从目标插槽的位置到起始插槽若存在晶圆，则选取第一片搜索到的晶圆，若不存在晶圆，则从目标插槽的位置到结束插槽搜索第一片存在的晶圆。
- 可设置搜索插槽位置的上限/下限。
- 提供设置规则需要挑选的量测片数。
- 支持多种维度绑定规则，如不同的片数的批次，不同的产品的规则，不同参数的规则均可以设定。
# 机台限制
- (j-1) 机台限制可根据主机台和子机台（Tube、chamber等）设置控制条件: 产品/工艺流程/段/站点/配方/光罩/探针卡/晶圆片数量/批次， 所有控制条件需要支持可用通配符，所有控制条件可以单独设定也可组合设定。
- 基于相同的主机台和子机台允许设定多条。
- 可支持动态条件管控：批次可在特定时间区间内去限制最大可运行的晶圆片数或批次数。
- 可复制机台限制，并支持复制后修改。
- 可用已有的批次通过模拟器确认机台限制的设定正确与否。
- 需支持对于机台限制正反向的设定。
- 支持设定不同的层需要使用相同的设备。
- 需要支持优先级设定，后设定的优先级高于先设定机台限制条件。
- 支持基于产品、工艺流程、工序、模组、机台组、机台类型、机台、子机台进行查询筛选已设定的机台限制规则，至少有一个筛选条件需要输入值。
- 批次流片被机台限制卡控住时需要提示被哪些条机台限制卡控住，以及机台限制设定人信息。
- 预先验算批次在未来的指定站点是否在哪个机台/子机台被限定无法加工，用户必须给出明确单站点及机台/子机台，无需支持一次性多站点查询。
- 支持机台限制信息批量导入导出。
- MES自检机台限制设置是否正确（含格式）。
- 机台限制模拟：提供指定批次和指定机台是否存在机台限制的查询。
# 容许时间
- 支持提前设定容许时间( Q-Time)，允许设定最小时间、预警时间、最大时间，在各个时间可以设定暂停、发邮件、加急动作其一或多个动作并存；发邮件支持不同邮件组、收件人。
- 支持把定义好的容许时间(Q-Time)复用，关联到不同产品、工艺流程、工序进行复用。
- 修改工艺流程时如有变更站点之前有设定容许时间(Q-Time)需要一并更新相应容许时间(Q-Time)，如有无效容许时间(Q-Time)需提示给用户进行修改。
- 支持在实验分片单(SRC)与异常恢复单(RRC)中定义容许时间(Q-Time)以及继承原工艺流程上的容许时间(Q-Time)。
- 支持容许时间(Q-Time)创建、修改、复制，支持通过复制其他来生成新容许时间(Q-Time)，支持批量处理。
- 数据清除及存档：提供完整功能给终端用户以及IT进行容许时间(Q-Time)主数据及历史数据进行删除以及存档动作，该动作不能影响报表数据库，不能影响现有批次流片及查询。
- 批次出机台之后可以对多个容许时间(Q-Time)在同一时间点开始计时。
- 批次入机台之后可以对多个容许时间(Q-Time)在同一时间点结束，在取消入机台后可以对容许时间(Q-Time)继续计时。
- 当分支路径A上有容许时间(Q-Time)结束站点，但批次实际走了另外一条分支路径时，则需要有容许时间(Q-Time)自动结束机制。
- 容许时间(Q-Time)的时间条件达到时系统需要自动采取预设的动作，容许时间(Q-Time)触发机制需要合理，避免不生效、邮件爆发等异常问题。
- 当批次处在容许时间(Q-Time) 结束站点时，如果发生取消进站，或跳站动作，容许时间(Q-Time)需要继承回来.。
- 如果批次在加工过程中容许时间(Q-Time)超时，要能够在出站后触发批次扣留动作。
- 分批时，子批要能自动继承母批的容许时间(Q-Time)。
- 并批时，检查子母批的容许时间(Q-Time)， 要能自动将子批里面比母批短的容许时间(Q-Time)带回母批。
# 产品物料清单(BOM)管理
- (先要进行物料的分类维护（如主要材料、间接材料、辅料等），维护物料编码，从而工步配置对应的材料。
- 产品物料清单(BOM)管理，需对产品物料清单(BOM)进行分类管理（如返工、试制、量产、替代、设备维修等）、版本管理、状态、适用范围等管理，并与ERP中产品物料清单(BOM)管理功能有效配合、集成，可在产品族、产品、制造资源、批次等对象中定义产品物料清单(BOM)结构；对各个对象的 产品物料清单(BOM)进行变更追溯及变更履历记录。
- 消耗追溯及记录管理，订单及批次的产品物料清单(BOM)中可记录实际的物料消耗情况以及物料的数量、来源、批次、质量等级信息。
- 物料管理，需与ERP仓库管理模块集成共同完成线边仓、发料、检查、退库、物料容许时间(Q-Time)等业务。
- 可按工步设置、查询，版本及激活权限控制。
# 编码、代码模型管理
- 可基于业务需要，灵活配置定制包括但不限于产品批次（子母批）、工艺配方、产品、制造资源等对象的编号规则。
- 可基于业务需要，灵活配置各类原因代码（如报废代码、扣留代码、释放代码、终止代码等），并方便的查询、展示。
- 支持批量导入/导出，支持批量新增、修改、删除。
- 可根据业务需要，灵活配置各类形式的打印条码。
# 统一维护工具
- 支持查看全流程配置信息，包含产品、工艺路径、分支路径、工段、站点、机台组、机台ID、配方&设备工艺流程程序ID (PPID ,Process Program ID)、工程数据收集计划(EDC Plan)、光刻板、探针卡、容许时间(Q-Time)、污染、工时、模组、作业指南等信息。
- 支持基于一款产品、工艺流程所有相关数据（包含关联机台或机台组、分支路径、配方、工程数据收集计划(EDC Plan)、光刻板、探针卡、容许时间(Q-Time)、污染、工时、模组、作业指南等）的一键完整导入/导出功能。
- 支持批量导入/导出，导入/导出格式统一，能够将导出的数据修改后直接导入，导入时有比较与校验防错功能。
- 通过功能以及Excel模版导入的两种方式都需支持比较与校验防错：
- 新旧版本比较，并将不同点显著标识；
- 数据校验，避免用户误操作，提示用户数据错误信息。比如数据类型、长度、特殊字符、空值、值范围、依赖关系、唯一性、与系统其他数据相关性一致性、是否重复等校验。
- 各列要有栏位填写说明要求。
- 支持工艺制程信息变更签审后生效。
- 支持在工段特定工序的先后顺序校验。
- 工艺流程及工艺流程下各对象升版本时需要定位出影响范围：实验分片单(SRC)、容许时间（Qtime）、在制批次、先行批（Pilot），反馈用户进行处理。
- 导入完成后，可以基于此产品/工艺流程进行正常流片生产。
# 在制品管控
- MES批次为MES流片监控等操作主体对象晶圆批次，生产流片批次承接来自内外部生产订单，监控等则为库房领料用来做日常、机台保养、挡位用的晶圆批次。
- 支持创建各类批次，批次类型支持自定义，包含生产、工程、测试、控挡片、陪片等。
- 生产批次支持ERP订单下达时MES自动创建相应订单生产批次，也支持a手动创建生产批次，生产批次类型为生产、工程、测试：
- 自动创建：ERP订单下达MES能自动创建生产批次，接收ERP对生产批次产品、工艺流程、数量、收货库房、长文本、优先级、计划扣留、计划下线时间、计划入库时间、批次负责人、招标方等信息，并能根据批次命名规则及ERP订单类型自动确定批次类型；
- 手动创建：支持用户手动创建批次编号，并按一定规格校验防错；支持Excel批量导入创建批次，并校验防错。
- 需支持包括外延订单、晶圆订单、划片订单、晶圆复测订单，芯片复测订单在内各种订单的批次创建。
- 允许用户在批次还没有开工之前取消已创建的批次。
# 创建源批次
- 自动创建：支持ERP发料时MES自动创建相应物料源批次，源批次建立与生产订单批次关联，也支持手动创建源批次，源批次可选建立或不建立与生产订单批次关联。
- 手动创建：支持用户自动批次编号并关联到生产批次，系统创建时需按一定规格校验防错；支持Excel批量导入创建批次并关联到生产批次，并校验防错。
# 批次下线
- 绑定生产批次、源批次装载到载具中，如有源批次、生产批次有生产订单对应则通过生产订单关联，如无生产订单信息则通过产品与源产品对应关系匹配才能下线，对批次类型、载具类别进行对应管控。
- 支持打印履历卡、批次标签、流程卡，已有相关模版。
- 批次下线时需要支持记录晶圆片Wafer的背刻码（T7Code），背刻码伴随批次全生命周期。
# 批次与载具关联管理
- 一个载具可以容纳一个或多个批次，一个批次只能在一个载具内。
- 系统需要记录批次与载具的关联，晶圆片与插槽位置的关联。
# 批次查询
- 提供多种界面查询批次，可通过筛选条件过滤批次清单。
- 提供按设备，按站点查询批次。
- 查询批次界面提供基本在制品操作按钮。
- 查询条件包括：批次号，批次类型，批次优先级，位置，工作区域，载具号，批次状态，产品，工艺流程，子流程，工序段，设备组，设备，批次负责人等，支持定制。
- 显示要对特殊的批次高亮显示，如扣留的，有容许时间，优先级高的等。
- 批次未来作业的查询，可选择查看批次已触发的 容许时间，计划扣留，站点可用机台的信息。
- 批次/晶圆 的操作会留下历史，并提供详细方便的历史查找功能。批次/晶圆历史查询不能对生产造成影响。
# 批次进站出站
- 支持手动，自动模式。
- 支持进站（Track-In）检查一系列逻辑，如机台状态，污染，容许时间，机台限制等一系列逻辑。
- 允许的状况下可取消批次进站，批次状态会返回到进站之前。支持EAP端发起的取消批次进站。
- 可配置批次出站后批次停留在当站或下一站。
# 预约/取消预约
- 预约时，系统需要检查批次开工所必须通过的条件，如设备机限，配方管理(RMS)，污染等。
- 提供可设置预约载具的最大数量。
- 支持将批次预约到指定机台。
- 支持取消预约。
- 人工预约支持预约列表，支持排序。
# 扣留及释放
- 支持按照扣留原因代码（HoldCode）及角色进行批次扣留，用户可以被赋予多个角色，释放代码（ReleaseCode）与扣留原因代码（HoldCode）有对应关系表配置。
- 支持叠加扣留暂停，支持转扣留暂停的代码，支持更改备注。
- 扣留时支持发邮件通知相应角色里所有人员，默认自动发送对应角色人员，同时支持额外手动勾选具体人员。如扣留给光刻工艺工程师，则只需发送邮件给光刻工艺工程师。
- 支持所有状态下批次都可扣留，对应扣留类型不同。
- 每次释放仅能释放对应一条扣留。
- 支持扣留/释放原因代码可调整配置及查询。
- 加工状态的暂停(Running Hold)：支持因设备故障等原因将在加工中的批次扣留住。
- 批量扣留/释放。
- 支持根据不同筛选条件查询所有被扣留批次，批次已扣留时间、扣留代码、扣留人、释放责任角色，支持导出查询结果为Excel、CSV文件；
# 计划扣留/取消计划扣留
- 支持设定当站（还未入机台时）出机台扣留，未来站点入机台前及出机台计划扣留，计划扣留重复生效默认为F，批次未到站时可取消该计划扣留，支持批量设定及批量取消计划动作。
- 支持批次在当前的流程上，对未来某个站点设置计划扣留。
- 按生效时间点不同，计划扣留可以分为 进站就扣留(Pre Future Hold)和出站时扣留(Post Future Hold)。
- 批次的计划扣留支持批量预设扣留的设置。计划扣留的设置可以被更改，删除；支持批量设定及批量取消计划动作。
- 批次的划扣留生效后，与版本无关 (版本升级后计划扣留应该被继承)。
- 支持叠加计划扣留。
- 在取样站点设定计划扣留，批次即使非抽样批次也需要扣留在取样站点。
- 基于产品/工艺流程计划扣留某个站点设定计划扣留，即该产品达到该站点就被自动扣留。
- 在取消本设定的同时需要将已经受该规则影响批次的计划扣留需要一并取消，已经扣留批次则单独释放。
# 分批及合批
- 需要支持逻辑/物理分批，支持写分/合批备注。
- 支持一次分一个或多个子批，支持同一时间合并多个子批到母批。
- 分批时可同时设定计划合批站点或不合批，该计划合批动作可以被单独取消。
- 当分批时，子批默认继承母批的（包括正常产品批次、实验分片批、先行批、非生产批等批次分批）的基本属性、当前站点信息、上下流程、长文本、备注、容许时间、实验单、扣留、 计划扣留 、计划分批、污染等级。
- 同家族且相同产品相同工艺流程相同站点的子批可以互相合批，但合批原始（祖宗）批次不能被合并到子孙批中，有计划合批动作的批次不允许被合批，除非计划合批已到达合批当站或提前取消计划合批动作。
- 支持倒片机EAP自动化分、合批。
# BankIn/BankOut
- 对于停留在当站长时间不流的批次，可以将批次进行暂存（bank），由提出人将批次扣留给制造部且为暂存的专用扣留代码，然后由制造部部决定是执行暂存还是拒绝暂存转扣留代码给提出人部门角色，该动作权限必须赋予制造部制定人员；已暂存的批次可以执行暂存转出操作回到产线。
- 暂存位货架操作需要指定批次暂存到的目标边仓位置，便于后续查找暂存的实物批次。
- 在扣留批次中不显示暂存批次。
# 跳站及退站
- 跳站及退站（Skip/Reposition）时校验载具，批次和站点的污染等级，以及工艺工序重复跑货（Double Run）防控。
- 需支持量测机台跳站，权限可放宽至PE，各模组只能跳自己的站点。
# 报废及取消报废
- 支持基于晶圆报废/取消报废和整批报废/取消报废，取消报废时需要指定返回载具（原载具或新载具）进行分配，卡控污染等级。
- 基于晶圆报废可针对不同晶圆选定不同报废原因代码。
- 基于整批报废选定统一报废原因代码。
- 批次报废时影响到在该批次上设定的实验分片单(SRC)等单据（涉及未来站点确定晶圆）需要提前识别并告知单据设定人进行单据变更或取消重设，要求实验分片单(SRC)即使已经审批发布也能撤回进行修改。
- 如果一个载具多批次情况，则载具中所有批次须统一报废或取消报废。
- 取消报废后批次所有信息回退到报废前。
- 支持报废原因代码可调整配置及查询。
# Terminate/UnTerminate
- 支持整批终止或取消终止。
- 支持不同终止原因代码。
- 基于整批终止选定统一终止原因代码。
- 如果一个载具多批次情况，则载具中所有批次须统一终止或取消终止。
- 取消终止后批次所有信息回退到终止前。
- 支持终止原因代码可调整配置及查询。
# 入/出库
- 需支持晶圆入库、外延片入库、芯片入库等多种形式产品入库。
- 支持入库时，先由制造部入库，再由仓库人员进行操作入库确认，入库进度可以查询，入库信息可以通过接口抛转到ERP。
- 入库支持退库返回现场的操作。
# 键合(Bonding)
- 支持键合(Bonding)和解键(De-bonding)。
- 可配置器件批次主产品/封装批次辅产品批次产品对应关系，在键合站点选择主产品批次跟已经配对的辅产品批次进键合机台；出机台后，辅产品批将会被终止/走回收流程。支持手动/半自动操作模式。
- 支持多层键合/堆叠。
# 标签打印
- 便捷的标签制作工具，标签布局可以由用户制作和修改。
- 标签数据可以由IT配置，无需开发代码。
- 支持文字，图片，条码，二维码等。
- 支持打印预览和默认打印机设置和标签打印。
- 支持不同尺寸标签。
- 可将标签打印策略挂载在站点上，在批次进出站是打印相对应的标签。
- 可对接主流标签打印机。
# 包装管理
- 支持设置对应产品的包装标准文件，以及包装示范文件（扫描批次号自动调取该客户或产品的包装说明书）。
- 通过标签模块，可设置对应包装条码的格式。可对应多个包装标签，每个标签可支持多个打印条码（一维或二维）与内容。
- 包装工步，支持通过标签扫描二次比对功能。
- 可根据产品物料清单(BOM)以及产品数据，自动给出对应包装材料的数量、型号。
- 支持多层包装及包装ID管理。
# 线边仓管理
- 根据物料类型不同的管控要求，设置各线边仓位的物料属性。
- 收料管理：产线物料管理人员通过扫描物料标签进行收料确认，登记实际收料信息。
- 发料管理：根据生产计划要求，产线物料管理人员通过扫描物料标签进行发料确认。
- 批次管理：根据约定的规则形成批次管理策略，能根据批次实现物料追溯管理。
- 退库：剩余料的退库管理。
- 线边仓存储时效管理：支持对存储时间敏感的物料进行存储时效管控。存储时效异常时，扣留物料批次并提醒。
# 作业流程单管理
- 作业流程单管理模块要求将作业流程单按照业务需要关联各相关对象，支持按工单/按批打印，并记录打印历史。
- 作业流程单可通过MES系统在各工位显示相应的作业要求，并可以在投批时进行打印。
- 打印印记可以通过配置对应的图片打印在流程单上，供作业人员参考。
# 现场货架管理
- 支持创建货架，货架有货架ID、类型、区域、架位，行列数等属性。
- 货架存放内容有工装、批次成品、批次半成品等。
- 材料、批次上下货架都可以记录，并且有对应的代码选择。
# 先进封装管理(WLP)
- 支持晶圆先进封装(WLP)管理。
- 支持产品键合关系的配置，预先配置键合的工步位置。
- 支持多层键合关系。
- 键合工步限制必需选择了最少一组晶圆进行作业，也允许多片一起键合。
- 键合错误或者异常允许解绑重新作业。
- 键合的两片晶圆的Bin值支持叠加分析综合良率。
- 键合的两片晶圆的AOI结果值支持叠加分析综合良率。
- 键合之后的晶圆支持重新导入新的分选文件给分选机作业。
- 分选之后的晶圆经过人工检验，如有不良，系统支持手动打点(ink)去除。
- 系统可以支持统计打点(ink)之后的良率，追溯到颗粒级别。
- 生产过程的测试数据，系统可以解析。
# 批次科研调整(SRC)
- 实验分片单(SRC)设定
- 实验分片单(SRC)的设定内容应支持设置工序、机台、子机台、片号、配方、采集参数、采集规则、光刻板、探针卡、容许时间、曝光所需信息等，这些信息可以通过快速添加其他正常的站点做原始模版之后再进行变更调整，也可直接添加情景化路径模版（预先定义好的Rework flow）作为原始模版之后再进行变更调整。
- 采集规则：采集片数、片号，支持工程数据收集，识别管制计划来获取工程数据的的规格与控制线。
- 容许时间：实验分片单(SRC)内部子流程新增容许时间，支持关联原主流程上容许时间(Q-Time)设定，需要支持继承原有容许时间(Q-Time)起止站点设定，如果容许时间(Q-Time)的起点或终止站点（或起点和终止站点都）在分片单的分/合批站点之间，工艺未设置时需要提醒用户进行设定。
- 支持设定实验分片单的类型、原因、代码等。
- 设定时需要校验提醒是否与机台限制反向设定（不允许入机台）相违背，如有违背则需提醒用户进行修正。
- 设定时需检查机台/子机台上配方开关状态。
- 实验分片单可以配置采集工程数据参数，参数是否进SPC 管制计划，允许由用户自行配置。
- 支持设定起止站点。
- 用户可以针对批次创建当前站点或未来站点的实验分片单，用户可以这对一个批次开立多张实验分片单，但不允许同一站点开立多张实验分片单。
- 实验分片单不允许嵌套开立。
- 支持实验单过程批次设定计划扣留，计划扣留站点可以在实验单站点或终止站点。
- 支持快速复制已有实验分片单后进行修改调整创建，快速复制的实验分片单创建时必须进行所有逻辑校验，不限定必须相同产品或工艺流程，如果复制的实验分片单的起止站点当前批次不存在允许用户进行调整。
- 实验分片单创建完成后，可以有预设的签核流程进行签核，签核完成即可等待批次到实验分片单开始站点进行实验分片单执行，如果签核未完成，批次提前到达实验分片单开始站点，则自动对批次进行暂停等待签核完成。
- 批次执行实验分片单
- 可以针对已经有实验分片单的批次，在设置的触发站点分批，分批后，子批按照实验分片单的流程跟规格来进行实验子批的在制品作业。
- 当片数与实验分片单设定片数不符合时需提交创建人进行确认，该判断需要提前告知。
- 子批实验做完后，可以跟在设置站点等待的母批进行合批。
# 批次调整
- 支持批次在当前站点上进行产品，工艺流程的版本升级，或是改变产品，工艺流程。
- 支持生产批次在当前的流程上，对未来某个站点设置流程重新绑定，进行产品，流程的 版本升级，或是改变产品，流程。
# 异常恢复单(Recovery Run Card，RRC)
- 在生产工步加工过程中，批次或者设备发生异常，批次在生产状态下暂停(Running Hold)，可触发异常恢复单。
- 提供手动触发异常恢复处理单，开立流程可以开给工艺、设备进行签核。
- 触发异常处理单后，提供选择已经加工过的晶圆片，尚未加工的晶圆片和正在加工的 晶圆片进行分批后分别处理。
- 支持将未加工的晶圆片分出子批后，取消进站（Track In）
- 支持将已加工的晶圆片分出子批后，执行出站（Track Out）
- 支持将正在加工的晶圆片分出子批后，进入重工流程（Rework Flow)
- 支持选择部分晶圆片，设定临时作业步骤进行作业，设定内容应支持设置工序、机台、子机台、片号、配方、采集参数、采集规则、光刻板、探针卡、容许时间(Q-Time)、曝光所需信息。
- 异常恢复单执行时，可以分别对配置的不同情景的晶圆进行分批作业，作业完成再合批回到异常恢复单中配置的完成节点。
# 先行批试跑(Pi-Run)
- 产品先行试跑：为了验证批次情况，通常流程为分批前量、加工、后量合批，主要实现的是主动提示并要求分批，并连续加工多个工序后自动合批。
- 机台先行试跑：为了验证子机台情况，通常流程为分批前量、先行批加工验证、后量合批，主要实现只有先行批子批才能入要试跑的子腔加工，其他批次只能使用非试跑腔体及配方：
- 多腔体机台的子腔在PM之后都需要先行批试跑（Pi-Run），试跑合格后才放行。
- 先行批试跑（Pi-Run）子批在试跑先行站点只能进试跑子腔。由用户在设定先行批试跑（Pi-Run）单时指定哪几片晶圆进入哪个机台的哪个子腔加工，一个先行批试跑（Pi-Run）单可以允许同时设定多个先行批试跑（Pi-Run）子批。
- 批次先行试跑：主要有分批和合批设定，基本与先行批试跑机台相同，区别在于没有必须要入的先行试跑子腔。
- 先行试跑单子设定：
- 先行试跑批次具体跑多少片需要由用户（工艺指定）设定。
- 需要支持可提前设定先行试跑单。
- 一个批次生效的先行试跑单子只能一张，如有多个的需求，则先对批次做计划扣留，待当前的先行试跑单执行完成后再设定新的。
- 如果先行试跑单子设定的分批晶圆在到达分批站点前已经报废，则将单子提交给工艺进行再确认，工艺在再确认过程中可修订其他晶圆作为先行试跑子批。
- 不能将所有晶圆全部设定到先行试跑子批中，无论是单批还是多批。
- 先行试跑优先级高于静态采样，即先行试跑的子批经过采样站点时一定执行而不跳过，在采样站点的片数抽样规则按站点本身设定。
- 支持在合批站点设定计划扣留，在合批前扣留。
# 机台暖机(Season)
- 暖机控制对象包括主设备和子设备（Chamber）。
- MES在自动及手动模式下都需要支持暖机控制。
- 暖机批次管理包括状态模型，生命周期控制，属性维护，降级，清洗，回收等，暖机批降级后批次编号需要重新生成。
- 触发暖机条件包括：按时间（idle时间） 、按数量（按连续加工或累计加工晶圆片数量） 、按配方（更换或闲置时间）。
- 暖机条件设置
- 能够定义暖机计划，暖机批，暖机配方，系统应该支持按用户配置的信息做暖机。
- 按时间 (设备/配方闲置时间)；
- 按晶圆片数量 (持续/累计)；
- 按配方（组）更换；
- 用户维护暖机信息，如：查询，新增，修改，履历，复制等。
- 设备配方更换的暖机，支持并行批量（Batch）。
- 设备配方闲置的暖机， 支持并行批量（Batch）。
- 暖机可以是独立非生产片管理模式：只有暖机使用的非生产批单独进设备；也可与生产批一起进设备。
- 一个机台的不同子设备（Chamber）可以并行跑暖机。
- 当暖机出现某些异常，用户可以手动完成暖机或重置暖机状态及信息，以使得重新做暖机。
- 支持污染等级管控。
- 持多次暖机，即机台需要用多种暖机片跑多个程序才完成暖机要求，且有时效管控。
# 机台监控(Monitor)
- 控制对象包括主设备和子设备（Chamber）。
- 支持对设备机台的稳定性可靠性进行监控，根据时间，作业量的要求进行配置进行配置监控要求，设备监控量测的参数必需合格才允许设备或者腔体子设备继续作业。
- MES在半自动及手动模式下都需要支持监控控制。
- 需要提供页面给用户维护监控信息，如:查询，新增，修改，履历，复制等。
- 需要基于时间周期来设定监控执行内容及单据，可重复执行，并可设定在目标时间的有时间区间，并有提前通过邮件预警功能，超出时间则必须做监控。
- 支持自定义测机监控流程，监控流程支持作业和量测站点，支持设备监控参数的前后计算，支持一次测机，可以监控设备的多个参数。
- 支持定义设备和监控流程的对应关系，可以根据设备组，来定义整个设备组的测机流程。
- 设备测机监控流程中所使用的测机片产品、测机片数，循环测机的次数，测机片回收循环的次数军均可以配置管控。
- 因设备负荷或一些紧急情况，可以人工修改监控的间隔时间。
- 如果批次测量数据通过，则自动重置机台间隔时间。
- 监控批次的可以支援来自不同的源批次，监控批次的晶圆支持来自不同批次的多片晶圆。
- 监控可以是独立非生产批模式，只有监控使用的非生产批单独进设备，也可与生产批一起进设备。
- 监控批次使用完成后可以回母批、直接回收、废弃；监控批次在使用过程中量测值超标允许报废、回母批进行回收。
- 监控源批可以回收(Recycle)、降级(DownGrade)、报废(Scrap)，回收(Recycle)完成后将批次当前使用次数清零，并更新回收(Recycle)次数、总使用次数。监控批降级后批次编号需要重新生成。
- 支持使用EAP操作并自动收值。
- 支持污染等级管控。
# 挡片(Dummy)
- 支持挡片批次在炉管机台的进出控制。
- 无论设备是自动模式还是手工模式，都支持挡片批次放入和取出作业 。
- 晶圆使用计数、膜厚累加控制机制。
- 支持大小回收流程及次数控制。
- 支持污染等级管控。
# NPW Downgrade
- 用户可以自定义非生产片降级规则。
- 提供界面进行非生产片降级到其它非生产批的类型。
# 机台管理
- 普通设备
- 设备号支持自定义，建模期设备支持批量导入。
- 设备的类型、物理区域、工艺区域、资产编码、负责人等信息均可维护。
- 设备的最大加工批数、片数均可以管控。
- 校验载具，批次污染等级，通过才能开工。
- 支持一个载具中存在多个批次。
- 倒片机作业
- 多个装载端口（Load Port）的倒片机，如机台能力的情况允许下，支持 4个装载端口可以灵活组合，在装载端口数量允许的情况下，同时执行多个导片任务。
- 支持导片任务基本类型，如：载具更换，批次转移，分批，合批，批次下线（Wafer Start），批次入库（Wafer Out），转角，晶圆片ID验证，晶圆片排序，晶圆片翻转。
- 倒片机机台要能支持流程内 (Inline Sorter) 和非流程 (Adhoc Sorter)的载具作业。
- 支持批次的分批（1-N)，合批(N-1)。
- 倒片机装载端口（Load Port）限制：需要提供以载具/批次属性来限制装载端口（Load Port）的使用。
- 倒片机载具限制：提供设定来决定载具是否能传送到另一个载具。
- 在倒片机任务列表中的批次不能再被倒片机任务预约。
- 支持流程内倒片机站点，能够按照用户的设定完成导片动作。
- 支持原物料盒和载具之间的传递。
- 炉管控制
- 支持生产批次跟 炉管监控批次一起组批，也支持只有生产批次或是监控批次单独 组批的跑货模式。
- 炉管所用的监控晶圆片纳入非生产批管理，进行准备/使用/回收/降级（Preparation/In-Use/Recycle/Downgrade）的管理。
- 支持炉管监控批次在 In-Use 的作业中，可以进行前量/组批/后量。
- 在炉管 (Furnace) 作业，支持指定批量组合中的生产批次在炉管中的顺序(Boat Position)。
- 炉管批量批次中若有监控批次，支持其相应的监控片(Process Monitor Wafer ) 获取量测结果应用于（贴值）同一批量（Batch）的生产批次。
- 炉管之后生产批次需要在当站等待监控批次量测结果。
- 支持根据监控片所处的位置，对相应区域的产品批次采取动作，如果某一个监控片超规，只对其对应的批次采取动作。
- 支持炉管缓冲批管理，支持从 EAP 得到炉管缓冲批次剩余数，决定是否可以组批。
- 支持设定并行批量（Batch）的最少晶圆片数量，跑货时根据生产片数量决定配方。
- 支持炉管机台填充挡片（Dummy）的机制。
- 需要有最大预约并行批量（Batch）数的管控 (在这个炉管上可以预约的最大批量数)。
- 酸槽批量跑货(WET Batch Run)
- 可以选择同时跑2个载具或是 1个载具。
- 用户可设定并行批量（Batch）的最少晶圆片数量。
- 光刻机台
- 光刻 (Photo) 机台支持双光罩使用。
- 支持一个主机台同步控管两个子机台 (Tracker/Scanner)，并可设置两个子机台的配方(Tracker Recipe/Scanner Recipe)。
- 多腔体设备
- 支持多腔体设备并行/串行跑货模式。
- 可根据子机台的状态，下达给EAP最合理的 设备工艺流程程序ID (PPID ,Process Program ID)组合。
- 化学机械研磨(CMP)设备
- 开工前校验研磨垫(PAD)、抛光头(Head)的使用期限。
- 定义进出装载端口所允许的载具类型，并在开工前进行校验。
- 可定义进出装载端口的组合，如装载端口1(Port1)进，装载端口2(Port2)出，装载端口3(Port3)进，装载端口4(Port4)出。
- 量测设备
- 可将工程数据采集计划下达给EAP。
- EAP根据工程数据采集计划将量测数据上传至MES。
- 激光打码机台(Laser Mark)
- 支持用户根据客户，产品，订单等信息建模。
- 支持通过EAP自动打码。
# 晶圆探针检测(Wafer Sort/Probe)
- 持基于芯片探针测试（CP）机台测试结果文件一定规则解析获取工程量测数据，并支持基于该工程量测数据结果计算每片晶圆片合格管芯数更新到晶圆上。
- 支持基于量测CP测试结果文件按照一定规则叠加运算；并进行相应的批次扣留，异常片号自动分批扣留、报废等功能。
- 支持一定规则的卡控，例如片号不一致、晶圆标准颗粒数不一致等问题。
- 支持异常OCAP出发后正常片号分子批出站继续流片功能。
- 晶圆合格Die二次报废需求
- 可作为工序动作，也可临时专门操作，需要手动进行晶圆上Die报废，当一片的Die合格率或指定参数低于一定值时该片自动报废或扣留，此时批次可在产线正常流片。
- 需要历史记录。
- 支持随时打印晶圆合格Die数。
- 复测业务
- 支持对已入库批次重新投入产线，进行CP测试、或者外观检查、或者重新包装。
- 总要求：①复测订单下达发料；②复测流程模型；③复测操作Die报废；④质量追溯到源批计算良率。
- CP测试及外观检查需要支持更新良率数量（基于上次入库合格数再手动报废晶圆上Die数更新）
# FT测试
- FT后的BIN结果可以进行良率卡控。
- 在FT等待结果站点可以根据BIN结果自动进入下一步(扣留、放行等)。
- 在FT等待结果站点如果失败了可以扣留，工程师处置时候可以选择放行合格Die(拆批)，重工不合格Die(拆批)。也可以选择整批重测。
- 可手动通过MES 扣留/放行。
- 工程师可以对需要重测的部分选择站点(多个或一个)进行重工。
- 在FT步操作中显示返工次数(子批次从父批次复制)。如果超过返工计数，显示警告并保持。
- FT测试完成后，需要按BIN进行分批，合格Bin流通，不合格Bin 自动扣留等待处置。
- FT时候支持维护载带等耗材的对应关系，作业时进行校验，防止用错并对载带批次记录使用历史以供查询。
- FT做完后需要在下一个站点等待处置。
- FT后支持并批，需要能自定义并批规则(规则参数配置包括批次，产品名等)。
- FT零头并批需要支持先进先出，少的先推荐原则，由用户并批时候显示出满足条件的批次，不满足条件的也要能显示出原因。
- FT后支持自动拆批。
- 整批可以继续流通，零头留在零头仓库中。
- FT的不良品随良品整卷一起流通入库。
- 良品进良品仓，不良品进废品仓。
- FT支持在作业过程中部分出站功能。
# WAT
- MES可以设WAT站点。
- 当WAT测试站点出站之后，批次扣留在WAT 结果等待站点等待规则检验的结果。
- 结果判定后需要支持放行批次，不合格批次触发OCAP流程，无数据的部分需要手动或者自动发送要求去取得规则检验的结果。
# 产品测试Bin管理
- 可根据工厂管理需要，对指定的产品设置 Bin 等级以及Bin等级划分机制，以及不同Bin 等级对应的产品编码、加工工艺数据、处置方式（如降级、报废等）。
- 可设置Bin划分工序，在此工序完成后自动按照Bin 测试结果分级生成新的分级批次，并记录新分级批次与原批次的对应关系以及新分级批次中每个关键对象的ID 信息及来源信息，并按照设定的业务规则根据不同Bin 等级对应到不同的产品编码。
- 需可记录并根据Bin 等级自动完成批次拆分并进行对象、单位、产品编码的转换，拆分后的批次中需记录其中芯片的来源片号、批号、Bin等级等信息，并按照设置的业务规则及工艺要求，MES对不同Bin 等级的批次进行使用限制、适用范围限制并规定其后加工过程，比如根据Bin 等级规定并限制其可发往的工序、机台等。
- 系统可支持能根据产品、对象来源、Bin 等级，规定其入库及加工等方面的限制条件，例如可设定根据Bin 等级进行分级分批入库，不允许不同批次、不同bin 等级的芯片混合入库或混合发料，不允许不同Bin 等级的批次进行合批等规则。
- 可根据 Bin 等级，对产品进行降级或报废处理，并对应到相应的产品编码。
# 超规动作计划 (OCAP)
- 支持自定义不同超规动作计划(OCAP)流程，超规动作计划(OCAP)流程步骤支持用户方便的自行编制并部署上线。
- 可以在超规动作计划(OCAP)流程中定义超规动作计划(OCAP)功能/行为/操作，例如扣留批次/释放批次/重量/加量（片号）/转换机台状态/重工。
- 支持设置量测站点关联多个工艺站点，支持允许设置当量测站点数据超规格线(OOS)则将对应工艺机台扣留，也支持接收来自SPC系统传递扣留工艺机台要求时，将对应工艺机台扣留。
- 超规动作计划(OCAP)流程涉及批次需要返工处理的则需要开立异常恢复单(RRC)。
- 支持针对批次自动生成相关超规动作计划(OCAP)ID。
- 支持与外部SPC系统接口集成设定超控制线(OOC)时对批次设定扣留动作等及相应超规动作计划(OCAP)流程，并自动触发扣留批次、对应工艺机台以及自动生成超规动作计划(OCAP)单据。
- 集成SPC显示对应的管制图。
- 支持批次超控制线(OOC)/超规格线(OOS)时对批次设定扣留动作等及相应超规动作计划(OCAP)流程，并自动触发扣留批次、对应工艺机台以及自动生成超规动作计划(OCAP)单据。
- 产生超规动作计划(OCAP)后需要与质量异常单签核系统集成，并将批次信息（包含但不限于批次、段、站点、超规动作计划(OCAP)原因等）传递至外部签核系统，超规动作计划(OCAP)流程中的指定步骤（如判断重测是否超控制线(OOC)/超规格线(OOS)）需支持传递至签核系统。
# OQC
- 支持设置出货检查(OQC)站点，允许出货检查(OQC)人员在出货检查(OQC)站点为批次指定质量等级，按照晶圆等级进行分批入库。
- OQA AVI 站：如果发生超规格线(OOS)， 自动扣留批次。
- 进站、出站、包装。
- 可根据其他系统（如WAT / Defect /SPC/Lot Type 等）系统集成，检查批次是否符合出货条件。
- 出货条件可维护在MES或其它系统。
# 污染管理
- 在工艺流程上，每个站点都可以设置接受进站的污染类别，以及出站后需要转换的污染类别。
- 在批次派工之前，需要对载具与站点的污染类别进行检查，当发现不匹配时不允许进行派工。
- 当批次将进行另一种污染类型的制程时，必须进行载具交换；例如：一个批次当前的站点是 FE 的污染类别，而下一给站点是 Cu 的污染类别，则需要进行载具更换后才可以进行派工。
- 对于倒片机机台的污染类别管理，只有设置在污染类别对应表里的载具，才可以将晶圆片传到另一个载具。
- 进行实验分片单与异常恢复单的批次作业时，要提供站点的污染类别设置，提交时要进行污染类别检查，符合污染类别转换规则才允许其生效，也要按照污染类别管理的规则进行加工。
- 基于制程的污染级别，载具分为不同的污染类别。
- 对于批次进入机台和站点的污染等级对应关系的管控，批次出机台时更新为工序污染等级及载具类型。
- 对于批次重新定位(退站、跳站、调整工艺流程、调整站点等)进行污染等级的管控。
- 跳站、调整工艺流程、交换载具、分批、取消报废、取消终止等操作需要卡控污染等级匹配问题。
# 载具管理
- 支持用户创建载具，分配载具，释放载具，废弃载具，取消废弃载具，报废载具，取消报废载具，清洗载具，交换载具，调整载具。
- 载具管理提供查询载具具体信息，查询载具历史，查询载具状态的功能。
- 支持批次类型与载具种类关联对应。
- 支持污染管控。
- 需支持来料盒子、过程载具、送库房载具等各类载具管理。
- 命名规则可自定义批量创建，也可根据 自行创建，也可批量导入。
- 所有通过载具编码查询批次的功能都需要能通过批次编码查询。
- 对载具进行生命周期管理。
- 载具上有唯一码标签。
- 载具信息包含但不限于载具状态、模型、位置、最后位置、载具类型、清洗时间、载具种类、载具槽位信息等。
- 载具清洗需要通过对载具的模型设置，需要在站点中检查是否达到载具清洗次数或期限，当批次入/出机台时检查载具已达到载具清洗次数或期限时系统需要提示用户交换载具并清洗旧载具但不卡死本次入/出机台加工。
- 快捷查询需要清洗的载具以及当前存储位置。
- 若由于批次长期扣留导致载具超清洗周期，系统需支持自动更换载具并将过期载具送洗。
# 光罩管理
- 光罩命名方式可由用户定义。
- 光罩按类型，等级管理。
- 提供界面维护光罩信息，如：查询，新增，修改，履历。
- 根据业务需要定义光罩的属性，如：类型，目录， 设备类型，等级。
- 提供光罩相关状况变化，追踪及跳转控制，支持光罩状态管理。
- 光罩扣留可以多重扣留，可以针对不同扣留分别释放。
- 支持与 EAP 集成的光罩盒位置管理。
- 支持手动将光罩盒位置管理（移动）。
- 支持光罩和光罩盒清洗，检测等的状态控管。
- 用户可以设置检测的规格，对于光罩的检验能够收集数据，可通过 SPC 系统控制。
- 支持周期控管光罩检测。
- 支持周期控管光罩盒清洗。
- 作业时，需要检查光罩的晶圆数量和检测的时间。
- MES要能自动更新光罩状态，位置，晶圆片计数。
- 支持通过所选光罩决定配方。
- 检查光罩运行间隔控制（不能连续超时使用）。
- 支持光刻板工程量测数据采集，光刻板量测结果（颗粒）超标需要自动扣留光刻板。
- 提供光罩操作及其所使用需要记录历史，历史可查询。
- 光罩检测机台整合EAP 和 MES，MES能查询到光罩处理历史。
- 支持对光罩装载端口(通过配置管理) 进行管理；支持与EAP对接。
- 光罩盒（光罩Pod）管理：清洗时间/清除时间。
# 光阻管理
- 通过扫描光阻瓶身上的条形码，可以在系统生成光阻瓶的编码(该瓶光阻唯一标识)。
- 光阻退冰(Defrost)
- 光阻建账后自动开始退冰计时。
- 退冰时间达到后，自动更改光阻状态；并自动发送/显示预警。
- 默认优先使用退冰完成光阻； 也可手动选择。
- 光阻更换(PR Change)
- 光阻更换时，系统 check 用于更换的新老光阻是否符合要求，若不符合要求，则退回。
- 更换光刻胶时需检查同种型号领入时间，优先使用最先领入的光刻胶(先进先出原则)。
- 光阻更换完成后，自动更改两瓶光阻状态。
- 可更换单个管路光阻。
- 光阻超过有效期后自动扣留光阻。
- 光阻退冰开始，超过设定时间后，自动扣留光阻。
- 用于维护不同机台所需的光阻类型。
- 光阻在系统内状态的改变，均需产生履历。
- 用户在系统内的操作的作业，均需产生履历。
- 机台上光阻使用完毕后会有警示。
- 光阻使用过程可以强制结束。
- 光阻安全库存提前预警。
# 探针卡管理(Probe Card Management)
- 提供界面供用户对产品，站点，分组，测试程序与设置文件等设置对应关系。
- 可以接收探针卡信息来创建探针卡，并更新位置。
- 提供探针卡的分组。
- 在CP测试的时候，提供探针卡透过探针卡分组跟设备组来跟测试机台关联。
- 提供探针卡检查对应关系后可以装载/卸载到机台。
- 探针卡需要增加扎针次数属性，并进行寿命管理(EAP通过EAP接口，非EAP通过文件解析)；支持探针卡记录晶圆片计数。
- 批次进站时候需要校验有效期。
- 扎针次数与晶圆计数已达上限或超过需要限制使用。
- 如果晶圆计数剩余数量还没达到上限，但是少于本次作业的晶圆片数量则报警，由员工选择继续进站还是终止进站。
- 提供探针卡到达上限后，可以送修并切换状态。
- 提供探针卡送修后再回到可用的状态。
- 提供探针卡装载到机台的查询。
- 当CP 的实验分片单(SRC) 时也需要计算探针卡寿命。
- 提供探针卡状态切换的历史信息。

- 维保清单建立(PM设备检查清单)
- 支持设备检查清单模板配置功能（按设备模板或者按设备），支持系统默认的签核逻辑。
- 支持添加、删除以及修改设备检查清单步骤。
- 支持导出设备检查清单模板以及导入设备检查清单模板。
- 参数类型（设置必填以及选填参数）。
- 参数提示（未填写必输入参数，完成会进行报警提醒）。
- 规格限制以及检查（是否需要OOS上下线卡控）。
- 添加用户执行设备检查清单步骤时所需的共享附件（如执行PM所需要SOP文件或者图片）。

# 备品备件管理系统
- 支持填写配件基本信息进行领料，录入系统；支持导入。
- 提供产品默认的配件录入工具。
- 将配件组里面配件组装到 设备/子设备，将此时挂在 设备/子设备的配件进行卸载，并且选择处理的方法(废弃，等待维修，待装机，已拆换，清洗)。
- 配件替换支持非维保任务的替换以及维保任务的替换。
- 替换过程中显示当前配件的使用情况。
- 系统提供设置配件的最大循环量。
- 支持对于配件使用次数的统计，当用户将配件从设备/子设备或者子设备拿下来时候，配件对应的次数属性自动加1，并与配件循环使用规格进行比较。
- 当配件领入，会有一个唯一识别号，当配件领入到在线仓以后，可以唯一性的进行跟踪记录。
- 显示配件的相关信息以及执行的时间。
- 与报警系统(AMS)系统进行整合，系统对于配件的使用期限会进行定时的监控，如果发现配件逾期，会将相关信息发送给预警系统。
- 用户可以配置维保需要更换的配件清单。
- 支持维保对配件更换的卡控，如果未更换必更换的配件，维保任务无法完成。
- 提供用户建立配件组的界面，并且同时支持导入工具可以进行统一的创建。
- 提供用户界面可以给机台添加配件组，用户在进行配件上机的时候会从该组中找到可用的配件。
- 提供用户可以根据配件号，序列号，供应商序列号，模组，状态，位置等进行查询对应配件的数量。
- 用户可以根据查询出来的配件清单，用户可以根据导出功能把配件信息导出。
# 权限管理
- 人员任职周期管理。
- 重要的系统及其功能操作都要有权限验证，重要操作需要再次输入账号和密码。
- 按设备或设备组。
- 按用户或用户组增减设备或设备组操作权限。
- 按设备或设备组增减可操作的用户或用户组。
- 当用户对设备进行设置或动作时，需要根据用户设置对设备的操作权限.如：进站（Track-In），出站（Track Out），切换设备状态/模式等。
- 设备状态权限，需要提供界面维护该权限.即什么样的角色可以将设备状态从什么状态切到什么状态.。
- 提供管控产品/工艺流程建模维护权限。
