# 3 故障侦测
- 3.8 支持建模处配置条件时可选择用户加入自定义的信息标签(MES Context字段 (可与工艺绑定))
- 3.9 UVA激活时支持签核功能
- 3.10 依据Context设置规则筛选所需监控的Wafer以进行UVA模型设置，如First wafer monitoring
- 3.11 数据窗口可以通过Step，Recipe Time，Sensor规则，offset来划分
- 3.12 数据窗口可以通过连续多点上升下降规则划分
- 3.13 数据窗口的定义支持参数化。参数可以随着当前Run的Context变化而调整，例如窗口可以自动匹配当前Recipe的main step
- 3.14 提供Function使Rawdata通过计算生成虚拟Sensor，Function支持表达式方式，并可以通过客制新方法扩充aFunction
- 3.15 提供Function使Rawdata和指标值通过前后多点的运算生成Moving Average ， MovingRange 等虚拟Sensor，并支持无缝扩展
- 3.16 提供Rawdata的非线性监控
- 3.17 窗口内的任意Sensor支持灵活的过滤功能，可以通过设置过滤掉一些噪音数据
- 3.18 系统提供向导式的快速建模工具，并可以通过Sensor的历史数据建议使用的统计方法
- 3.19 支持连续多片wafer之间的模型的统计值计算，例如计算连续两片之间MovingRang，并在LotID切换的时候进行重置。统计算法应该包括Mean，Max，Min，Sum，Slope，EWMA
- 3.20 支持对工艺中的某个步骤的时间进行累加，在超过规格之后可以报警并复原
- 3.21 Sensor的DataQuality在检查不合格的情况下，可以跳过部分模型的监测
- 3.22 提供在长时间的制程及时检查规格的解决方案，避免在制程结束时才发现问题
- 3.23 窗口支持对Recipe的任意一段区间进行监控
- 3.24 提供在cycle step 中批量检查main sub-step 的解决方案, 支持将cycle step 的所有main sub-step 设定成一个窗口,用一个规格.
- 3.25 支持发现工艺时间异常的问题，例如Wafer工艺过程中由于一些原因没有上报End事件，FDC系统可以经过配置及时发现并通过OCAP报警
- 3.26 支持设置Fixed spec
- 3.27 支持设置Delta spec，即在Target的基础上进行加减运算生成spec
- 3.28 可以根据设备(tool),腔室(chamber),配方(recipe),recipe step,或其他Context信息(product,stage)设定规格
- 3.29 模型的报警等级分为Warning，Alarm，Outlier
- 3.30 每个报警等级可以设置不同的规格和对应的OCAP
- 3.31 支持批量使用历史数据对所有的腔室生成规格
- 3.32 生成规格时选择的run list支持抽样
- 3.33 自动计算规格的时候，能够个别指定生成warning limit ,Alarm limit, Outlier limit的上下限对应sigma的倍数，sigma算法可以选择和自定义
- 3.34 每一个Indicator都可以单独设定计算规格时的训练样本组：PerProcessType、PerTool、PerChamber每次计算LIMIT都保留记录并且可以查看每次计算所用样本
- 3.35 内置5种sigma算法，可以给不同的模型选择适合的算法。算法列表：PSEUDO Sigma，Bounded Boxplot Sigma，SIMR2, SROBUST, SUMVU
- 3.36 规格种类支持N-Σ(目标值 +- N-Σ), Δ(目标值 +-Δ), 删除百分比(目标值 +- 删除*N%), 移动平均目标值(用户定义N值).
- 3.37 target bias(目标值偏移)支持product、tool、chamber维度设置offset值
- 3.38 具有50种基本的计算公式，平均绝对偏差AAD,Area，Count，Cp，Cpk，Dt，FirstPoint，HitCountAbove，HitCountBelow，Intercept，IQR, Kurtosis，LastPoint，MAD,Max，MaxDownwardSpike，MaxGap，MaxJump，MaxOscillation，MaxSpike，MaxUpwardSpike，Mean，MeanG，MeanH，MeanT，Median，Min，MinGap，Mode，Percentile，PointCount，PointCountAbove，PointCountBelow，PointCountBounded，Q1,Q3,Range，RMS，SampleRat，SampleTime，Skew，Slope，SlopeR2，Stdev，StdevT，Sum，TimeCountAbove，TimeCountBelow，TimeRange，Trend
- 3.39 支持MeanAgg算法拓展算法计算两组平均值之间的差值
- 3.40 通过集成AMS系统支持多种报警方式 ，如Email/SMS/Phone/Wechat
- 3.41 通过集成MES支持HoldLot/HoldTool/HoldChamber/HoldRecipe
- 3.42 支持设置MES/EQP Chamber之间的Mapping关系
- 3.43 支持Model算法的二次开发，提供sample code
- 3.44 FDC需要卡target以上百分比的规格 (例如, target以上10%的部分才卡)
- 3.45 模型的规格支持复制粘贴，
- 3.46 支持SPC Alarm rules ，支持自定义Alarm rule
- 3.47 支持基于算法、参数设定Alarm rule模版
- 3.48 支持Alarm rule的仿真测试，用以评估新的Alarm rule的执行结果
- 3.49 模型支持设置多个Alarm rule
- 3.50 模型可以设置级别，高级别的模型报警后，低级别的模型不再报警
- 3.51 支持MVA多变量分析
- 3.52 AI集成，支持利用AI算法建立监控模型
